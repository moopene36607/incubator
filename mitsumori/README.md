# mitsumori (見積もり)

**日本の町工場のための、小ロット見積書 AI 自動生成ツール。**

仕様メモ + レートシートを入れるだけで、日本のビジネス文体に沿った正式な見積書 (税抜・税込・特記事項・適格請求書発行事業者登録番号入り) を 30 秒で出力する。

---

## 痛点

日本の中小製造業 (約 43 万社、5 人以下が約 6 割) で見積もり業務は今も「Excel に詳しい古株 1 人の頭の中」に依存している。

> **「一番怖いのが『属人化』で『〇〇さんしか触れない Excel』というのがメーカーに多数存在し、マクロ職人のような人間が組んだ関数で仕事が回っているが、休んだりすると誰も Excel 触れないなどありえない環境です。」**
> — ITmedia/kn 2025 年中小製造業実態調査

> **「見積作成で属人化を防ぐ方法として、社内での業務マニュアルの整備が非常に重要だが、多くの中小製造業では担当者の感覚や経験で価格が決まっている。」**
> — ESTman 公式ブログ (2024)

具体的な業務上の痛み:

- 同じ部品でも担当者が違えば見積金額が ±20% ぶれる
- 古株が休むと、新人が見積もり 1 件に半日かかる
- 顧客が要求する「日本語ビジネス文体の正式見積書」を作るには敬語と業界慣習を熟知する必要があり、若手は怖くて出せない
- インボイス制度施行後 (2023/10) は「適格請求書発行事業者登録番号」「税抜・税込両方」の表示が必須となり、既存の Excel テンプレートが軒並み崩壊
- 精密加工従事者は 2000 年 25 万人 → 2020 年 15 万人と大幅減少、若手継承が困難

## 既存ツールが解決しない理由

| 既存ツール | なぜ町工場の課題を解けないか |
|----|----|
| **CADDi** | 大手・中堅メーカー向け、3D CAD 図面連携が前提。1〜5 人町工場は対象外。 |
| **meviy (ミスミ)** | 自社調達向け。外部顧客への見積書を発行する用途には使えない。 |
| **ARUM (ARUMCODE)** | NC 加工コード自動生成。商業見積書 (顧客向け文書) は対象外。 |
| **ESTman** | 見積フォーマット管理は可、AI で日本語ビジネス文章 + 特記事項を生成する機能はない。 |
| **Excel テンプレート** | 属人化の温床、インボイス対応で全部作り直し。 |

**ギャップ**: 「仕様メモ → 30 秒で日本語ビジネス文体の正式見積書」 を、月 ¥4,000〜¥10,000 で提供する SaaS は確認できなかった。これがこの prototype が埋めようとしているスペース。

## mitsumori が何をするか

```
仕様メモ JSON
(部品 / 材質 / 数量 / 工程 / 各種レート)
        │
        ▼
  pricing.py 純粋関数で計算
  (材料費 + 加工費 + 段取り按分 + 利益率)
        │
        ▼
  Claude API が日本語ビジネス文体で
  特記事項を生成 (材料相場備考、量産単価提案、
  検査成績書、運送費取り扱い等)
        │
        ▼
  御見積書 markdown
  (税抜/税込・適格請求書登録番号・代表者印枠付き)
```

3 つのコンポーネント:

1. **`pricing.py`** — 純粋関数のレートシート計算。各社が自社レートを config で差し替えるだけ。テスト容易。
2. **見積書フォーマット** — 日本のビジネス慣習に沿ったレイアウト (件名 → 御見積金額 → 明細 → 内訳 → 取引条件 → 特記事項 → 発行者)。インボイス制度対応 (T 番号、税抜/税込併記)。
3. **AI 特記事項生成** — Claude が案件情報をもとに、材料相場注記・量産時単価変動・検査基準・梱包仕様などをビジネス敬体で 5〜6 行生成。これが新人が一番書けない部分。

## 動かす

### AI を使わないモード (API key 不要)

```bash
python3 mitsumori.py samples/sample_input.json --no-ai
```

`samples/sample_input.json` は SUS304 ブラケット 100 個 + 補強プレート 200 個の小ロット案件 (典型的な町工場のケース)。出力は完全な見積書 (特記事項のみ AI 生成項目なので省略される)。

### AI fallback モード (リッチな特記事項付き)

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python3 mitsumori.py samples/sample_input.json --out quote.md
```

特記事項のみ Claude に投げる (1 案件 1 回の API 呼び出し、1 件約 ¢1)。

`examples/sample_output.md` に AI 特記事項込みの完成形を予生成済み。

## 検証済み smoke test

- ✅ 単部品計算: 100 円/個 × 10 + 加工 5 分 × ¥100/分 × 10 + 段取 ¥5,000 + 利益 20% → 税抜 ¥13,200, 税込 ¥14,520
- ✅ 段取り費は数量比例で複数 line に按分される
- ✅ ¥100 単位での単価丸め (町工場の慣習)
- ✅ 数量 0 / 空 items 入力で適切に ValueError
- ✅ Sample 入力 (300 個・2 line item) → ¥315,700 (税抜) / ¥347,270 (税込) 整合確認

## プロジェクト構成

```
mitsumori/
├── README.md
├── mitsumori.py             # CLI 主程式
├── pricing.py               # 純粋関数 価格計算ロジック
├── samples/
│   └── sample_input.json    # SUS304 板金加工 case
├── examples/
│   └── sample_output.md     # AI 特記事項込みの完成見積書
└── requirements.txt
```

総計約 350 行 Python。依存は `anthropic` のみ (AI 特記事項を使う場合のみ)。

## 真正版に必要だが prototype には未実装

- **PDF 出力 (印鑑枠付き)** — 印刷して FAX する文化が町工場に残る。WeasyPrint or ReportLab で実装。
- **CAD/PDF 図面の OCR 入力** — 図面から部品名・寸法・材質を Claude Vision で自動抽出し、JSON 入力を省略する。
- **顧客マスタ + 案件履歴** — 同じ顧客への再見積、過去案件からの転用。SQLite で十分。
- **マシンチャージ表 (時間帯・機械別)** — レーザー切断機 ¥180/分、NCT ¥130/分、ベンダー ¥140/分などの細分化。
- **インボイス制度関連の自動チェック** — 顧客側の T 番号有効性検証 (国税庁公開 API)。
- **見積→注文→納品書→請求書の連携** — 受注確定したら同じデータから他文書を派生生成。
- **量産単価の自動提案** — 「100 個でこの価格、500 個ならこの価格」を 1 リクエストで複数シナリオ提示。
- **競合価格との乖離アラート** — 業界平均 (CADDi 公開データ等) と比較して大幅乖離時に注意喚起。

## 商業モデル

| プラン | 料金 | 対象 |
|----|----|----|
| **Free** | 月 5 件まで | お試し |
| **Solo** | **¥3,980/月** (件数 50/月) | 1〜3 人町工場 |
| **Standard** | **¥9,800/月** (無制限・チーム 5 名) | 5〜20 人規模 |
| **年払い** | 上記 × 10 ヶ月分 | 2 ヶ月分割引 |
| **Enterprise** | 個別見積 | 工業協同組合・地域事業承継支援団体経由の一括導入 |

**競合価格**: ESTman ¥5,500〜/月 → 機能が上回れば価格優位。CADDi は導入費 + 月額 ¥10 万円台で、別市場。

**TAM**: 日本の中小製造業 43 万社 × 月 ¥5,000 平均 = 月 ¥215 億の理論上限。1% 浸透 = 月 ¥2.15 億 ARR。Solo dev でも 5,000 社 × ¥4,000 = 月 ¥2,000 万の MRR を視野に入れられる。
