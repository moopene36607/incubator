### Round 46 — `expsense/` (Taiwan 🇹🇼 — SME employee reimbursement anomaly detection, **thirty-sixth AI pattern — Isolation Forest**)

- **題目**: 台灣中小企業 (5-30 人 SME) 員工費用報銷 isolation forest 異常偵測(每月 30-200 筆 reimbursement → 多維 features [amount_log / amount_vs_personal_cat / hour_atypical / is_weekend / category_idx] → 純函式 Liu, Ting & Zhou 2008 Isolation Forest ensemble → anomaly score 0-1 + top-K 可疑筆 + feature contribution → AI 寫「對帳建議 + 員工確認開場句 + 風險」)
- **解決的問題**: 台灣 5-30 人 SME ~30,000 家 + 50 人以下 ~80,000 家。老闆每月看 30-200 筆 reimbursement 花 **1-2 小時對帳**, 大部分時間在算「這筆合理嗎」而非找異常 → 常漏看多維異常 (異常高金額 + 異常時段 + 週末 + 重複報銷 + mis-categorize)。FB「中小企業老闆交流」5-10 萬人「員工報銷單看到麻木」每週重複問。鼎新 / 數位通 ERP 只做記帳結帳**不做 anomaly detection**;Concur / SAP Expense 月費 NT$30K+ 中小企業負擔不起;會計師事務所 NT$5-10K/月 only 看分類不看異常;Excel 找不出多維異常;ChatGPT 一次貼 100 筆不能跑 IF 沒持續比對。**Gap 結構性**:Isolation Forest (Liu et al. 2008) 學術成熟 17 年**沒人做成台灣 SME 可用 SaaS**。Google「台灣 報銷 isolation forest」零中文 SaaS。架構:**Isolation Forest** — `iforest.py` 100% 純函式 stdlib(random + math + dataclass);`harmonic(n)` 調和數 (前 100 精確/後用 ln+γ 漸近);`c_factor(n) = 2·H(n-1) - 2(n-1)/n` 不成功 BST 搜尋平均路徑長;`build_tree` 隨機 feat × 隨機 split 遞迴至 height_limit = ⌈log₂(sample_size)⌉;`path_length(point, node)` 落葉路徑長度 + adjustment for incomplete branch;`fit_iforest` ensemble n_trees 棵樹各 subsample;`anomaly_score(x) = 2^(-E[h(x)]/c(sample_size))` 0-1 分數;`top_k_anomalies` 排序 + threshold filter;`feature_contribution(x)` 每 feature 隔離力 (近 root = 隔離力強)。LLM 只寫「每筆對帳建議 + 跨員工模式 + 風險」**永不算 anomaly score / feature contribution**。Sample: 海星科技 5 員工 109 筆 5 月份報銷(注入 7 個 anomalies),**IF top-5 命中 4 個 inject**(R0106 餐費 NT$4,800 凌晨 23 時 / R0105 差旅雜支 NT$8,500 / R0108 交通凌晨 3 時 NT$880 / R0103 客戶招待 NT$18,800),老闆當月對帳壓到 30 秒。16 edge tests passed (harmonic basic + large n / c_factor monotonic / tree leaf for single point + zero variance / height limit respected / path length nonneg / score in [0,1] / outlier scores higher than inlier / top_k sorted desc / threshold filter / feature contribution returns all features / deterministic / small sample handled)。
