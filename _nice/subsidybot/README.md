# subsidybot(補助通)

**台灣中小企業 / 創業 / 個人補助 RAG Q&A 助手 — 30 秒比對你符合的政府補助。**

把「我這種狀況到底能申請哪些補助?」「青創貸款和微型創業鳳凰差在哪?」這類痛點,從每次自己 Google 兩小時(還可能漏)壓到 30 秒。

---

## 痛點

台灣政府補助 / 貸款資源**遍地都是,但散在 N 個機關**:經濟部、勞動部、農業部、數位發展部、各縣市政府、各中央二級機關...每年公告 + 截止 + 修改條件,普通創業者 / 微型企業主 / 個人 *根本不知道從哪裡查起*。

> **「請問台灣創業補助有哪些?文件要準備什麼?每次都要自己去 N 個不同網站找資料。」** — PTT Entrepreneur 2024-11

> **「青創貸款和微型創業鳳凰有什麼差?誰可以說明?」** — PTT ask 2025-01

> **「補助申請資格寫得像天書,根本看不懂自己符不符合。」** — Dcard 創業版 2024-09

具體痛點:

- 1-10 人微型企業主 / 個人創業者 *估計約 150,000 人*,大多沒有專屬會計師
- 每次查補助:跨 N 個機關 → 看 PDF 條件文 → 比對自己資格 → **2-3 小時/次,還可能漏**
- 補助有截止日,錯過就要等明年(SBIR / SIIR 等限定徵件期)
- ChatGPT 直接問:**幻覺率高**(常給已截止的方案)、**不知道最新公告**(知識截止日問題)
- 律師 / 會計師諮詢一次 NT$3,000-5,000,小型企業負擔不起

## 為什麼現在沒有對的工具

| 既有工具 | 為什麼填不了這個洞 |
|----------|-------------------|
| **SBIR 官網 / 創業台灣 startup.taiwan** | 靜態 FAQ,**無對話式查詢、無資格自動比對** |
| **104 政府補助資料庫** | 資料庫型,**無法回答「我符合嗎?」** |
| **ChatGPT 直接問** | 知識截止日後的公告不知道 + **幻覺嚴重**(金額 / 截止日常錯)+ 沒有引用 |
| **各縣市 LINE Bot** | 只管自己縣市,**FAQ 固定腳本** |
| **律師 / 會計師** | NT$3,000-5,000/次,小企業負擔不起 |

**Gap 結構性**:沒有任何工具能做到「跨機關 + 即時公告 + 對話式資格確認 + 引用透明 + 幻覺安全」。資料來源全部 *公開可合法爬取*,但沒人有動力做這個橋接層。

## subsidybot 在做什麼

```
用戶 profile (年齡 / 性別 / 已登記 / 設立年限 / 員工 / 行業)
+ 自由文字問題
        │
        ▼
  ┌──────────────────────────────────┐
  │ retrieval.py 純函式條件比對         │
  │  - 硬條件:年齡 / 已登記 / 員工數    │
  │  - 軟條件:行業 / 性別(含特殊規則)  │
  │  - 分類:fully / soft / ineligible  │
  └────────────────┬─────────────────┘
                   │
                   ▼
  ┌──────────────────────────────────┐
  │ Claude 答覆(嚴格 RAG corpus)     │
  │  - 只 cite retrieval 給的方案     │
  │  - 不編造 / 不幻覺                │
  │  - 必引用官網 + last_updated      │
  │  - 必標「以官方公告為準」         │
  └────────────────┬─────────────────┘
                   │
                   ▼
        對話式答覆 markdown
        (用戶可對照官網直接申請)
```

3 個關鍵架構決策:

1. **`subsidies_db.py` 結構化 corpus**:8 個代表性補助方案,每個含主管機關 / 金額 / 利率 / 硬條件(age_min / age_max / business_age / employee_max / business_required)+ 軟條件(industries / genders)+ 必備文件 + 官網 + last_updated。產品化會擴充到 50+ 並接政府開放資料 API 每日 sync。
2. **`retrieval.py` 純函式比對**:條件比對 100% 純函式,LLM 永不負責「你符不符合」這個判斷 — 避免 LLM 對年齡 / 員工數 / 設立年限這類數字 hallucination。LLM 只在 retrieval 結果之上做語意整合。
3. **嚴格 RAG**:System prompt 明令「絕對只 cite 提供的 corpus,不在 corpus 中的方案、金額、條件 — **絕對不能提**」。每個答覆必引用方案官網 + 必標 last_updated + 必加「以官方公告為準」免責 — 規避 ChatGPT 級的幻覺風險。

## 動作

### 純條件比對(免 API key)

```bash
python3 subsidybot.py samples/sample_query.json --no-ai
```

`samples/sample_query.json` 是 32 歲女、創業 1.5 年、3 人服務業的情境。--no-ai 模式只列方案匹配清單(不寫對話答覆),適合快速看「我符合哪些」。

### AI 對話式答覆(需 API key)

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python3 subsidybot.py samples/sample_query.json --out answer.md
```

Claude 接收 retrieval 結果 + 用戶問題 → 撰寫對話式答覆,包含:推薦排序、為什麼符合、申請文件、官網連結、申請時機提醒、不適合方案的原因說明。

### 預先產出的 demo

`examples/sample_output.md` 是完整答覆,展示真實諮詢場景:
- 32 歲女創業 1.5 年的線上手作課平台
- 主推 3 方案(數位雲端 + SIIR + 女性飛雁)各附理由 / 文件 / 官網 / last_updated
- 額外提供 2 個貸款選項(微型鳳凰 + 青創)
- 明確說明 2 個不適合方案的原因(CITD 行業 / 農村青年行業)
- 結尾標準免責「以官方公告為準」

## 已驗證 smoke test

- ✅ Sample case (32 女 服務業 創業 1.5 年): 6 個 fully_eligible / 2 個 soft_match
- ✅ 50 歲男 (45+) 符合 MICRO_PHOENIX(性別特殊規則「男(45歲以上)」)
- ✅ 25 歲男無公司:MICRO_PHOENIX 性別不符 + 年齡 ok = soft;YOUTH_LOAN 未登記 = soft
- ✅ 70 歲女:超過 MICRO_PHOENIX 65 歲上限 → soft(差一點)
- ✅ 30 歲農民:RURAL_YOUTH 完全符合
- ✅ Empty profile:industry-restricted 方案進入 soft(不靜默判 ineligible)
- 🐛 修了一個 bug:原 `_check_gender` 用 substring 比對,「男 in 男(45歲以上)」誤判為符合 — 改 exact match + 特殊規則處理

## 真正產品要有但 prototype 沒做

- **政府開放資料 API 自動 sync**:目前 corpus 是手寫 8 個方案,真正產品要每日抓 startup.taiwan.gov.tw + sbir.org.tw + smes.org.tw + 各縣市政府網站 → 自動更新 corpus
- **vector RAG**:目前是純結構化 + LLM,真正產品需要 RAG over 整份 PDF 公告(以解析複雜資格)
- **截止日 LINE 推播**:用戶設定關注方案 → 截止日前 14/7/3 天主動推播
- **比對助理**:把不同方案 side-by-side 比較(青創 vs 微型鳳凰差什麼)
- **資格自動更新**:用戶員工從 5 → 15 → 30 人時,系統提醒「現在多了哪些方案,失去哪些方案」
- **多語**:新住民 / 外籍創業者用英文 / 越南文問
- **完成度追蹤**:用戶申請 5 個方案,系統 dashboard 追蹤進度 / 文件清單
- **顧問 / 會計師媒合**:複雜方案需要顧問代辦,subsidybot 推薦合作會計師事務所
- **橫移其他 RAG 主題**:勞基法 Q&A / 健保給付查詢 / 農委會法規 — 同一框架擴充

## 商業模式

| 方案 | 月費 | 對象 |
|------|------|------|
| **Free** | 3 問 / 天 | 一般查詢者 |
| **Pro 個人** | **NT$299 / 月** | 創業者 / 微型企業主 (無限問 + 截止日 LINE 推播) |
| **Pro 企業** | NT$799 / 月 | 中小企業 (含 5 員工帳號 + 多方案進度追蹤) |
| **API access** | NT$0.5 / call | 會計師事務所、顧問公司、政府單位嵌入 |
| **Enterprise** | NT$5,000+ / 月 | 大型會計師事務所、顧問公司多人 license |

### WTP 計算

- 1-10 人微型企業主每月查補助 1-2 次 × 每次 2-3 小時 × 機會成本 NT$300/hr = **每月省 NT$600-1,800**
- vs Pro NT$299/月 = **2-6x ROI**
- 成功申請 1 個 NT$25 萬數位雲端補助 = **回本 70 個月**(再算上其他工具的時間)

### TAM

- 1-10 人微型企業主 ~150,000 人 + 個人創業者 + 自由業 = 估計 200,000+ 潛在用戶
- 取 1% 滲透 = 2,000 paying Pro × NT$299 = **月 NT$60 萬 MRR / 年 NT$720 萬 ARR**
- 加 API access(會計師事務所)+ Enterprise + 橫移 RAG 主題(勞基法 / 健保 / 農委會) = 翻倍以上

## 風險評估

| 風險 | 評估 | 緩解 |
|------|------|------|
| **政府推官方版** | 中 | 政府網站開發慢、UX 差;搶先建立 PMF |
| **AI 引用過時 / 已截止方案** | 高 | 每日 sync corpus;每個方案標 last_updated;答覆必加「以官方公告為準」 |
| **AI 編造資格條件導致用戶誤申請** | 高 | 系統 prompt 嚴禁編造 + retrieval 純函式硬比對(LLM 不負責資格判斷)+ 引用官網 |
| **資料源變動 / API 限制** | 中 | 政府開放資料保證持續供應;若 API 變,corpus structure 解耦可快速更新 |
| **複雜案件需要會計師** | 低 | 定位「初步比對」工具 + 推薦合作會計師,不做專業判斷 |
| **市場不願付費(覺得政府工具該免費)** | 中 | Free tier 3 問/天 + Pro NT$299 衝動消費價;成功申請 1 個補助回本多年 |
