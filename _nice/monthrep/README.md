# monthrep(月報先生)

**台灣才藝班 / 補習班老師專用 — 學生月報自動產生器。**

老師月底花 30 秒勾選 + 寫 3 句觀察,30 秒內生出一份家長願意讀完的繁中月報(LINE 友好版 + 正式 PDF 雙版本)。

把每月底「20-30 份學生評語打到半夜」的痛點,壓成一個晚上 30 分鐘搞定。

---

## 痛點

台灣才藝班 / 補習班老師每月最大時間漏斗,**不是備課,是月底寫評語**。

> **「每個月月底都要寫 20 幾份學生評語,一個晚上都在打字,根本沒時間備課。」**
> — PTT C_Education 板,常見討論模式

> **「才藝班老師說 monthly report 是最大的 admin 負擔,寫到懷疑人生。」**
> — Dcard 教師版

> **「月報 / 聯絡簿是週期性熱議話題,每逢月底必有老師抱怨。」**
> — FB「台灣補習班主、才藝教室經營交流」社團

具體業務上的痛:

- 1 位老師通常帶 20-30 位學生,每月底要寫 20-30 份個人化月報
- 平均 10-15 分鐘/份 = **每月 3.5-7 小時純文書時間**
- 寫得太空洞家長不滿意(「都複製貼上」),寫得太用心又花太多時間
- 才藝班月報需求結構固定(出席 / 進度 / 進步 / 加強 / 下月計畫),但每位學生細節不同,沒辦法用「複製貼上」解決
- 老師時薪 NT$400-1,000,**這個 admin 時間沒有人付錢**,完全是純成本

## 為什麼現在沒有對的工具

| 既有工具 | 為什麼填不了這個洞 |
|----------|-------------------|
| **補教王 / EduBase / 補習達人** | 排課收費點名系統,**完全沒有 AI 文字生成** |
| **ClassSwift** | 課堂互動工具,非月報 |
| **均一 / PaGamO** | 學生端刷題平台,非老師端 |
| **ChatGPT 直接問** | 老師不知道怎麼 prompt;每次要重打學生資料;沒有月份累積追蹤;沒有 LINE 友好輸出 |
| **Word / Google Docs** | 老師現況。每份手寫,屬人化嚴重 |

**Gap 結構性**:補教軟體商核心客戶是大型連鎖,沒動力做小型才藝工作室的「AI 文字生成」小功能。1-5 人才藝工作室是 VC 嫌小、補教軟體商嫌雜的縫隙。

## monthrep 在做什麼

```
老師勾選表單(本月主題 / 進步 / 加強 / 給家長的話)
        │
        ▼
  ┌──────────────────────────────────┐
  │ 純函式組裝月報標頭 + 出席統計      │
  │ 純函式產 LINE 友好純文字版         │
  └────────────────┬─────────────────┘
                   │
                   ▼
  ┌──────────────────────────────────┐
  │ Claude(系統 prompt + 科目 / 年齡  │
  │ band grounding) 撰寫 4 段月報內容 │
  │  ① 本月學習概況                   │
  │  ② 觀察到的進步                   │
  │  ③ 需要加強的地方                 │
  │  ④ 給家長的話 + 下月學習計畫      │
  └────────────────┬─────────────────┘
                   │
                   ▼
        markdown 月報草稿
        + LINE 友好純文字版
        (老師審閱 → LINE 群發 / PDF 寄送)
```

3 個關鍵架構決策:

1. **科目 grounding**:8 種主流才藝 / 學科(鋼琴 / 美術 / 英文 / 數學 / 舞蹈 / 小提琴 / 圍棋 / 作文)各有「典型評論面向」與「家長最在意的點」清單,讓 AI 不會寫成「您的孩子超棒!」這種空泛廢話。
2. **年齡 band 切換口吻**:學齡前 / 國小 / 國中 / 高中 4 種口吻自動切換 — 國中家長要看「分數量化 + 弱點分析」,學齡前家長要看「快樂學習 + 社交觀察」,完全不同。
3. **AI 嚴格規定不編造**:沒有提到的進步 / 比賽 / 出席日數**絕不能補**,system prompt 明確禁止。確保月報內容老師背得起。

## 動作

### 不開 AI(骨架版,免 API key)

```bash
python3 monthrep.py samples/sample_input.json --no-ai
```

骨架版包含完整月報結構(學生資訊 / 出席統計 / 本月主題 / 4 段 placeholder),適合 demo 也可以拿來給老師當作回填模板。

### AI 模式(需 API key)

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...

# 產 markdown 月報 + LINE 純文字版
python3 monthrep.py samples/sample_input.json \
    --out reports/04_lin_xiaoyu.md \
    --out-line reports/04_lin_xiaoyu_line.txt
```

AI 自動填 4 段內容,根據:
- 學生年齡層自動切換口吻(本範例為國小四年級 → 溫暖 + 具體舉例)
- 科目 grounding(本範例為鋼琴 → 評論視奏 / 拍子 / 雙手協調 / 比賽準備)
- 老師輸入的進步 / 加強 / 家長疑慮 → 對應段落

### 預先產出的 demo

`examples/sample_output.md` 是完整月報草稿,展示一份 4 月鋼琴才藝班月報(學生:林小雨 / 國小四年級 / 鋼琴 6 個月進度):
- 出席滿勤 4/4、作業完成率 70%
- 5 個具體進步點(視奏速度 60→75 BPM、小奏鳴曲完整度、比賽佳作、音樂表達層次)
- 3 個需加強(左手獨立性、練琴頻率、長樂句呼吸感)各附老師下月處理方法 + 家長可如何協助
- 個人化回應家長提的「該不該往音樂班發展」疑慮

## 真正產品要有但 prototype 沒做

- **學生資料庫**:每位學生長期累積本月進度 → 自動帶入下月「上月遺留待加強事項」
- **批次模式**:老師一次勾選 20 份學生資料,一次產 20 份月報(夜跑 30 分鐘搞定)
- **LINE 群發整合**:月報直接從工具一鍵發送到家長 LINE 群組(不需要老師手動複製貼上)
- **PDF 公版**:配合教室 logo / 老師簽名圖檔產製質感 PDF
- **語音輸入**:老師講 30 秒,Whisper + GPT 變成結構化勾選資料
- **家長端反饋**:家長收到月報後可一鍵回覆「了解 / 想討論」
- **學習趨勢圖表**:6 個月累積後自動產出視覺化趨勢
- **比賽 / 檢定 / 段考重要里程碑提醒**:從月報資料推斷下次重要日期

## 商業模式

| 方案 | 月費 | 對象 |
|------|------|------|
| **Free** | 月 3 份免費 | 試用 |
| **Solo** | **NT$299 / 月** | 1 人才藝工作室 / 兼職老師 |
| **Studio** | NT$799 / 月 | 5 人以下小教室,共享模板 + 學生資料庫 |
| **Chain** | NT$2,499 / 月 | 5-15 人連鎖補習班 |
| **年付** | 上方 × 10 個月 | 省 2 個月 |

### WTP 計算

- 老師時薪 NT$400-1,000(才藝班均偏向中位 NT$600)
- 每月 20 份月報 × 12 分鐘/份 = **每月 4 小時 admin × NT$600 = NT$2,400 機會成本**
- monthrep 把 12 分鐘壓到 2 分鐘 → 月省 NT$2,000+
- Solo NT$299/月 vs 月省 NT$2,000+ = **6.7x ROI**

### TAM

- 全台才藝班 + 補習班(教育部統計)約 50,000 家
- 取 2% 滲透率 = 1,000 個 paying user × NT$299 = **月 NT$30 萬 MRR / 年 NT$360 萬 ARR**
- 加上 Studio + Chain 高階方案、跨類擴大到家教 / 健身教練 → TAM x3+

## 風險評估

| 風險 | 評估 | 緩解 |
|------|------|------|
| **補教王 / EduBase 自己加 AI 月報** | 中 | 補教軟體商核心是排課收費,文字生成不是其 DNA;先佔小型才藝市場 |
| **AI 寫月報太通用、家長一看就知道** | 中~高 | 科目 + 年齡 band grounding + 老師勾選 5+ 具體觀察點 = AI 無從泛泛廢話 |
| **AI 編造老師沒提到的進步** | 高 | system prompt 明確禁止編造,測試多 case 確認;老師審閱 final gate |
| **老師覺得「我自己寫比較好」** | 中 | 第一個月免費 / 30 份 trial,讓老師親身體驗節省的時間 |
| **教育部 / 政府補貼 出免費版** | 低 | 政府工具開發慢,且通常只到「範本下載」這層 |
| **家長個資**(老師可能輸入學生姓名) | 中 | 月報以代稱選項;不長期儲存學生資料(stateless API call);有需要可走 self-host |
