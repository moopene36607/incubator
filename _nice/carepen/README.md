# carepen(照護筆)

**台灣長照居服員語音逐字稿 → LTCIS 服務記錄草稿 AI 助手**

居服員講一段話 → 30 秒內變成衛福部 LTCIS 規格的結構化服務記錄,把每天 1–2 小時的文書時間壓縮成 10 分鐘的「確認」。

---

## 痛點

台灣長照產業最大的灰犀牛不是缺人,是 **居服員每天花在「寫服務記錄」的時間越來越多**。

> **「評鑑還是一場文書整理大賽。」**
> — 台灣居家服務策略聯盟 公聽會發言(報導者 2024.03 整理)

> **「居服員把它寫成日記式流水帳,督導需要大量修改。」**
> — Care724 居服員社群討論

> **「代寫評鑑文件市場規模數萬至數十萬元。」**
> — 居盟理事長 涂心寧

具體業務上的痛:

- 每位居服員每天 4–8 個個案,每個個案結案要寫服務記錄,而 LTCIS(長照支付制度資訊系統)規定的欄位多、格式硬:服務代碼、計量、詳述、個案狀況、生命徵象、異常事件、後續建議。
- 居服員多為 50–65 歲、台語/國語混用、書寫能力參差,「日記式流水帳」就是現況。
- 督導每天花 2–3 小時改文書,本職的個案訪視管理被壓縮。
- **長照 3.0(2026 年上路)文書指標從 5 項增至 7 項**,行政負擔將進一步加劇。
- 評鑑年代寫產業真的存在(數萬到數十萬元/單位),代表「現有工具完全不夠用」是被市場用真金白銀背書的事實。

## 既有工具為什麼不夠

| 既有工具 | 為什麼填不了這個洞 |
|----------|--------------------|
| **衛福部 LTCIS** | 是申報系統,只有空白表單。居服員依然要自己組字。 |
| **仁寶 i 照護** | 大型雲端系統,主打照管雲/派案,不解決「文書產出」核心痛點。 |
| **Vocol.ai / 通用語音轉文字** | 不知道 LTCIS 服務代碼,不會輸出衛福部要的格式。 |
| **代寫產業** | 以人月計算,規模不上雲,單位/個別居服員一輩子沒進入 SaaS 市場。 |
| **Excel / 紙本 / LINE 群組** | 督導手改的源頭就是這些工具。 |

**Gap**: 沒有一個工具做「居服員 30 秒語音 → LTCIS 服務代碼自動分類 + 結構化記錄草稿」。這個縫隙在 2026 年長照 3.0 上路後會被市場放大。

## carepen 在做什麼

```
居服員手機錄音(30 秒~3 分鐘口語)
        │
        ▼
  Whisper 繁中 ASR(本 prototype 用文字逐字稿模擬)
        │
        ▼
  Claude(LTCIS 系統 prompt + 12 個常用服務代碼)
        │
        ▼
  結構化 JSON:
   - service_lines: [代碼, 計量, 詳述]
   - case_status_observations
   - vital_signs (體溫/血壓/脈搏/血糖)
   - abnormal_events
   - follow_up_notes
        │
        ▼
  純函式渲染 → LTCIS 服務記錄草稿 markdown
  (居服員確認 → 督導複核 → 上傳 LTCIS)
```

3 個關鍵架構決策:

1. **`ltcis_codes.py`** — 12 個 LTCIS 居家服務代碼當 seed dictionary。實際 100+ 代碼擴充只是加 entry,系統設計不變。
2. **AI 只做語意理解,從不算錢/編資料** — 體溫/血壓只能從逐字稿擷取,沒提到就用 null,**絕對不編造**(這是醫療紀錄硬規矩)。
3. **草稿 → 確認 → 複核 三道關** — 永遠定位「AI 是助理,人是把關者」,規避代寫產業的灰色責任問題。

## 動作

### 不需 API key — 從預先結構化 JSON 渲染

```bash
python3 carepen.py --structured samples/sample_structured.json
```

`samples/sample_structured.json` 模擬 AI 已經結構化好的結果,展示完整 LTCIS 記錄草稿格式。

### AI 模式 — 從逐字稿產生(需 API key)

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python3 carepen.py \
  --transcript samples/sample_transcript.txt \
  --case samples/case_metadata.json \
  --out output.md
```

逐字稿是模擬居服員口語(「今天早上九點到林阿嬤家做服務,進去的時候她在床上看電視。先量了血壓 145 over 92 比昨天高一點點 體溫 36.8 還 OK...」),AI 自動拆解成 7 個服務代碼 + 生命徵象 + 後續建議。

### 預先產出的 demo

`examples/sample_output.md` 已預先產出,無 API key 也看得到完整 7 行服務記錄(GA01 / BA10 / BA04 / BA13 / DA02 / BA05 / BA09)+ 血壓 145/92 已標警示 + 後續建議分 3 點。

## 專案結構

```
carepen/
├── README.md
├── carepen.py             # CLI 主程式(--transcript / --structured)
├── ltcis_codes.py         # LTCIS 服務代碼 seed dictionary
├── samples/
│   ├── sample_transcript.txt    # 居服員口語逐字稿(2.5 小時訪視)
│   ├── case_metadata.json       # 個案資料 + 居服員身分

└── requirements.txt
```

總計約 350 行 Python。依賴僅 `anthropic`(--transcript 模式才需要)。

## 真正產品需要但 prototype 沒做

- **Whisper 繁中 ASR 整合** — 真正產品要一鍵錄音 → ASR → 結構化,目前 prototype 仍需文字稿。可以接 OpenAI Whisper API 或本地 faster-whisper(離線、保護個資)
- **完整 LTCIS 100+ 代碼表** — 與某居服單位 BD 後拿到完整對照表 + 描述模板
- **直接送 LTCIS 系統 API** — 跳過手動上傳步驟(衛福部 LTCIS 應該有上傳 API,需洽談)
- **督導複核 workflow** — 居服員確認 → 督導改 → 簽核 → 鎖定的多角色流程
- **LINE 整合** — 居服員大多用 LINE 工作,直接在 LINE 群組錄音/接收紀錄草稿
- **個案 trend 自動 surface** — 連續 3 日血壓>140/90 自動標記、體重連 7 日下降提醒督導
- **離線模式** — 中南部偏鄉訊號不穩,需離線錄音 + 訊號回來再同步
- **多語/方言** — 居服員台語/國語混用,Whisper 對台語偵測還在改善中

## 商業模式

| 方案 | 月費 | 對象 |
|------|------|------|
| **Trial** | NT$0 / 30 天 | 試用 |
| **Solo** | NT$899 / 月 | 1–5 名居服員的家族經營小單位 |
| **Standard** | **NT$4,900 / 月** | 20–50 名居服員的中型居服單位 |
| **Multi-site** | NT$11,800 / 月 | 多營業所、跨縣市單位 |
| **政府補助版** | 由縣市政府/衛福部 AI 落地實證計畫支付 | 入門用 |

### WTP 推算

- 督導每月改文書時間 60 小時 × NT$300/hr = **每月 NT$18,000 督導成本**
- carepen 把它壓到 15 小時 = 省 NT$13,500 → NT$4,900/月一個月就回本
- 代寫評鑑文件市場一單 NT$30,000–80,000 → 把這個費用挪到 SaaS 上有 6+ 個月空間

### TAM

- 全台居服單位 1,400+,Standard 月費平均 NT$4,900
- 1,400 × NT$4,900 = 月 NT$686 萬全市場
- 取 5% 滲透率 = **月 NT$34 萬 MRR / 年 NT$410 萬 ARR**(solo dev 維生足夠)
- 加上長照 3.0 政策驅動 + 縣市政府補助計畫 = TAM 持續放大

## 風險評估

| 風險 | 評估 | 緩解 |
|------|------|------|
| **衛福部 / 仁寶推內建 AI** | 中~高 | 政府系統開發慢、仁寶現在主打派案;搶占居服員端心占率 |
| **AI 幻覺生命徵象** | 高 | system prompt 強制「沒提到=null,絕不編造」,人工確認永遠是 final gate |
| **個資 / HIPAA 風險** | 中~高 | 個案編號、姓名以代稱;Anthropic 不訓練資料;企業版可走 self-host LLM(LLaMA / Qwen) |
| **代寫產業反彈** | 低 | 我們不取代真實服務,只壓縮文書時間;代寫業最終會自己採用工具 |
| **居服員年齡層 IT 抗拒** | 中 | LINE 整合(他們本來就用)+ 督導 onboarding 訓練;不是要他們學新軟體 |
| **長照 3.0 規格再變** | 中 | 純函式設計,代碼/欄位常數改一行;訂閱客戶 lock-in 在「規則改我們先更新」 |
