## 純函式判讀

- **最關鍵 driver**: `歷史失約次數` (貢獻 37%)
  - 這診所的 no-show 主要決定於這個特徵, 改善這方面可以最有效降低 no-show

- **行動建議優先順序**:
  - 🔴 高風險 (P ≥ 60%): 雙重預約 + 預收訂金 + 24h 前 LINE 個人化確認
  - 🟠 中高 (P 40-60%): 前一天 LINE 提醒 + 預備候補名單 1-2 人
  - 🟡 中 (P 25-40%): 前一天 SMS 提醒
  - 🟢 低 (P < 25%): 標準流程

## ⚠️ GBDT 模型假設與限制

- **In-sample accuracy 高 = overfit 風險**: 真實 launch 必須 train/test split + cross-validation
- **歷史只 200 件不足**: 鼓勵診所累積 ≥ 500 件才正式上線預測;Pro 版加 transfer learning
- **季節 / 節日 / 特殊事件未納入**: 連假前後 / 颱風天 / 流感季節等 systematic 因素需另外處理
- **GBDT 容易 overfit on noise**: max_depth=3 + learning_rate=0.1 + 50 trees 是 conservative, 可降低過擬合
- **GBDT 不解釋為何**: feature_importance 給總體 driver, 個別 case 為何高需要 SHAP / TreeSHAP (Pro 版)
- **隱私敏感**: 病患就診紀錄涉個資 + 醫療性質, 雲端版需加密 + 院方同意 + 資料去識別化

## 🤖 AI 醫療管理顧問建議

陳醫師,**這診所的 no-show 真正關鍵 = 病患「歷史失約紀錄」(貢獻 37%)** — 過去爽約 2 次的病患很大機率再次爽約。其次是「忠誠度」(就診次數 + 距上次來訪),老病患很少 flake。雨機率 (8%) 跟預約時段 (6%) 是次要因素。整體 34.5% no-show rate 算偏高 (業界平均 15-25%),有結構性問題需要動 SOP。

**3 個立即可做的流程改造**:
**(1) 預約系統加「失約過 2 次以上者強制預收 NT$500 訂金」** — 高失約史病患 60-90% 願意付小額訂金保留位 (按時來就到付從療程金額扣抵), 直接過濾掉真正的 no-show 慣犯;
**(2) LINE 機器人 24h 前自動推播個人化提醒** — 高風險病患個性化「您預約明天 16:00 / 預估療程 60 分鐘 / 鳳鳴會空一節給您 / 改期請按此」, 比簡訊提醒回應率高 3-5 倍;
**(3) 建立「秒填候補」名單** — 5-7 個願意當天空檔被 call 的客戶 (通常是想做但時間難約的老病患), no-show 發生時 30 分內可填補, 減少醫師空班損失。

**本週 20 個 appt 行動分配**: 2 個 🟠 中高風險 (A1006 / A1003 都 3x 失約史 + lead time 長) **立刻打電話人工確認 + 啟動候補名單**;13 個 🟡 中等風險 → 前一天 LINE 推播自動化即可;5 個 🟢 低風險 → 標準流程不打擾。
