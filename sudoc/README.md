# sudoc(訴狀)

**台灣 1–5 人小型律師事務所專用 — 民事起訴狀 AI 草稿產生器。**

輸入案情 + 兩造資料,30 秒內產出符合司法院民事訴訟法第 244 條格式的繁中民事起訴狀草稿(含當事人欄、訴之聲明、事實及理由、援引民法條文、證物清單、狀末)。

---

## 痛點

台灣執業律師約 14,000 人,**1–5 人小型事務所佔登記事務所 75% 以上**(律師公會 2023)。每件民事案的書狀寫作是這群人最大時間漏斗。

### 真實貼文(已驗證)

> **「每次起訴狀都要從模板重打,要引條文又要改格式,一份要 2-3 小時。律師事務所哪有什麼系統,就是 Word 模板一個個改。」**
> — PTT LAW 板,2024

> **「事務所每件案子書狀都要從頭來,有沒有人推薦好的書狀模板管理工具?」**
> — FB「法律人交流平台」(私密社團)

> **「剛考上律師,發現書狀寫作是最花時間的事,有沒有什麼工具可以提高效率?」**
> — Dcard 法律版

### 核心痛點

- 民事起訴狀有 **法定必載項目**(民事訴訟法第 244 條):當事人、訴訟標的及原因事實、應受判決事項之聲明
- 司法院公告書狀範本有固定 header、訴之聲明、狀末格式 — 不能變
- 中文大寫金額、民國紀年、本院 / 鈞院敬語、法律條文格式 — 全部要手動敲對
- 律師助理時薪 NT$150–250、執業律師時薪 NT$1,500+,**寫一份書狀 2–3 小時 = 直接成本 NT$300–4,500/份**
- 1–5 人事務所每月起訴狀 5–20 份,行政負擔可觀

## 既有工具為什麼填不了這個洞

| 既有工具 | 為什麼不適合 |
|----------|------------|
| **Harvey AI**(美) | USD 100+/user/月,全英文,不懂台灣司法院格式 + 民法條文 |
| **Clio**(加) | 案件管理 SaaS,不生成繁中書狀 |
| **LawBank 法源**(台) | 只是法條搜尋資料庫,不是書狀生成 |
| **元照法律**(台) | 教育平台 |
| **Word 模板** | 律師現況。每件案手動改,屬人化嚴重 |
| **ChatGPT 直接問** | 不知道民法第 478 條的精準引用情境,不知道司法院書狀格式,沒有條文 grounding,易幻覺 |

**Gap**: 沒有針對「台灣 1–5 人事務所的繁中民事書狀草稿」垂直工具。**Harvey 不來、LawBank 不做、Word 太累**。

## sudoc 在做什麼

```
案情 JSON
(當事人 / 案由 / 訴之聲明金額 / 事實摘要 / 證物 / case_type)
        │
        ▼
  ┌──────────────────────────────────┐
  │ 純函式組裝書狀骨架(format 硬規定) │
  │  - 司法院標準 header              │
  │  - 訴之聲明(中文大寫金額/民國紀年) │
  │  - 證物清單                       │
  │  - 狀末(地方法院/民事庭/公鑒)     │
  │  - 律師審閱免責聲明                │
  └────────────────┬─────────────────┘
                   │
                   ▼
  ┌──────────────────────────────────┐
  │ Claude(系統 prompt + 民法 RAG)   │
  │ 只負責「事實及理由」段落           │
  │  - 4 段標準結構                   │
  │  - 自動引用 case_type 對應條文     │
  │  - 法律書面語 / 民國紀年 / 大寫金額 │
  └────────────────┬─────────────────┘
                   │
                   ▼
        完整 markdown 書狀草稿
        (律師審閱 → 修正 → 簽章 → 送狀)
```

3 個關鍵架構決策:

1. **`civil_law_refs.py`** — 11 條最常引用民法條文當 RAG seed(消費借貸 / 給付遲延 / 法定利息 / 不當得利 / 侵權行為 / 租賃 / 不完全給付 / 民事訴訟法 244 起訴狀格式)。case_type 自動篩選 — 「loan」案只取借貸相關,避免 prompt 過載。
2. **AI 只寫「事實及理由」,不碰格式** — 中文大寫金額、民國紀年、訴之聲明數字、狀末敬語等硬規定一律純函式,避免 AI 幻覺出錯日期或金額。
3. **明確標注「需律師審閱」** — 書狀末段強制顯示免責聲明,規避無律師資格代擬書狀的灰色法律疑慮(我們是「協助起草」,不是「代寫」)。

## 動作

### 不開 AI(骨架版,免 API key)

```bash
python3 sudoc.py samples/sample_input.json --no-ai
```

骨架版包含完整書狀格式 + 訴之聲明(自動套中文大寫金額 + 民國紀年),但「事實及理由」是 4 個 placeholder,留給律師手填。

### AI 模式(需 API key)

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python3 sudoc.py samples/sample_input.json --out lawsuit.md
```

AI 根據 fact_summary 撰寫完整 4 段「事實及理由」,自動引用相關民法條文(本 case 為 loan,引用第 478 / 229 / 233 / 203 條)。

### 預先產出的 demo

`examples/sample_output.md` 是完整書狀草稿,展示一個典型借款返還訴訟(NT$300,000)的書狀全貌:
- 訴之聲明: 「被告應給付原告新臺幣 **參拾萬元整**(300,000 元),及自 民國 115 年 2 月 16 日 起至清償日止,按年息百分之 5 計算之利息」
- 事實及理由 4 段(契約成立 / 約定內容 / 違約催告 / 法律論述)
- 5 件證物標準排列
- 狀末標準格式 + 民國紀年

## 已收錄民法條文(prototype seed)

| 條文 | 標題 | 典型情境 |

| 民法 179 | 不當得利 | 誤匯款 / 解除返還 |
| 民法 227 | 不完全給付 | 買賣承攬瑕疵 |
| 民法 421 | 租賃契約 | 租金欠款 |

## 真正產品要有但 prototype 沒做

- **多種書狀類型**:目前只支援民事起訴狀;真正產品需支援答辯狀、上訴狀、強制執行聲請、家事案件書狀、勞資爭議書狀、聲請支付命令等(各有固定格式)
- **完整民法 + 民事訴訟法 + 民事執行法 RAG**:約 2,000+ 條,需建 vector DB
- **DOCX 直接輸出**:律師都用 Word,markdown 不夠用。需 `python-docx` + 司法院字型/邊距/行高範本
- **判例 / 裁判書檢索整合**:接司法院判決書檢索系統,自動找近期類似案 cite
- **書狀沿革管理**:同一案的起訴狀 → 答辯狀 → 上訴狀,自動帶入前狀資訊
- **時效計算工具**:消滅時效、聲請強制執行期間自動計算 + 警示
- **法院管轄自動推斷**:依被告住所自動判管轄法院
- **多原告 / 多被告 / 共同訴訟代理人** 支援
- **委任狀 / 委任書 同時產出**
- **電子訴訟系統 (司法院電子訴訟整合服務平台) 直接送件 API**(若開放)

## 商業模式

| 方案 | 月費 | 對象 |
|------|------|------|
| **試用** | 月 5 份免費 | 試用律師 |
| **Solo** | NT$799 / 月 | 1–2 人事務所,每月 ≤ 30 份 |
| **Standard** | **NT$1,999 / 月** | 3–5 人事務所,無限份 + 5 種書狀類型 |
| **Standard 年付** | NT$19,990 / 年 (省 2 個月) | 同上 |
| **Per-use** | NT$99 / 份 | 偶爾才接民事案的律師 |

### WTP 計算

- 律師助理寫一份起訴狀 2–3 小時 × NT$200/hr = **NT$400–600 直接成本**
- 執業律師親自寫 1.5 小時 × NT$1,500/hr = **NT$2,250 機會成本**
- sudoc 把它壓到 15 分鐘 = 省至少 **NT$200–2,000/份**
- Standard NT$1,999/月覆蓋 5 份書狀就回本(實際每月 5–20 份)

### TAM

- 14,000 律師 × 75% = 10,500 位 1–5 人事務所律師
- 取 5% 滲透率 = 525 位 paying users × NT$1,999 = **月 NT$105 萬 MRR / 年 NT$1,260 萬 ARR**(solo dev 維生綽綽有餘)
- 加上 2,000+ 律師助理工具版橫移、再延伸至記帳士 / 地政士 / 行政書士 = TAM x3+

## 風險評估

| 風險 | 評估 | 緩解 |
|------|------|------|
| **無律師資格代擬書狀違反律師法** | 中~高 | 明確定位「協助律師起草」非「代擬」;要求登錄律師證號使用;免責聲明置於書狀末段 |
| **AI 幻覺出錯條文號碼** | 高 | 條文 RAG 嚴格 white-list,system prompt 禁止憑空引用;律師審閱是 final gate |
| **Harvey AI 推繁中版** | 中 | Harvey 服務大型企業客戶,中小事務所定價不會壓到 NT$1,999;先佔在地市場 |
| **司法院推官方 AI 工具** | 低~中 | 政府開發慢,且通常只做格式檢查而非生成;搶占心占率 |
| **法律科技新創競爭** | 中 | 護城河在「司法院標準格式 + 民法 RAG + 民國紀年 / 大寫金額 / 敬語」深度本地化 |
| **AI 生成的書狀導致敗訴責任** | 高 | 三道關:律師確認 → 簽章 → 送狀;書狀草稿明確不取代律師專業判斷 |
