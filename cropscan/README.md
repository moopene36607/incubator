# cropscan(田間衛士)

**台灣作物 AI 病蟲害辨識 + 台灣許可農藥建議。**

農民上傳作物 + 病徵描述 → AI 從台灣本土 corpus 比對 Top 3 候選病害 → 純函式輸出嚴重度、用藥、安全期、預防、農改場聯絡。

---

## 痛點

台灣 77 萬農家,中小型蔬果農農場(1-5 公頃)發生病蟲害時:

> **「葉子這樣是什麼病?用什麼藥?農改場打過去都在忙。」**
> — 農業易 FB 社團

> **「番茄葉子黃了,我上傳圖給農改場 LINE 群,3 天才回。」**
> — PTT Agriculture

具體痛點:

- **農改場推廣員人力有限**:每位平均覆蓋上百戶農民,LINE 回覆 3-5 天
- **農藥誤噴每次損失 NT$3,000-8,000**(噴錯藥沒效 + 又得重噴 + 殘留問題)
- **錯過防治黃金期**:病蟲害發現後 24 小時內處理 vs 一週後處理,損失可能差 10x
- **海外工具不認識台灣作物**:PlantVillage / Plantix 主要訓練美洲 / 歐洲 / 印度作物,台灣特有(蓮霧 / 鳳梨釋迦 / 火龍果 / 芒果炭疽)幾乎無資料
- **農委會「農業易」App** 只有圖鑑瀏覽,**沒有 AI 辨識**

## 為什麼現在沒有對的工具

| 既有工具 | 為什麼不行 |
|----------|-----------|
| **農改場 LINE 群組** | 人工回覆 3-5 天,黃金處理期已過 |
| **PlantVillage / Plantix** | 英文 / 訓練集偏歐美印,**台灣本土作物覆蓋極低** |
| **農委會「農業易」App** | 純圖鑑瀏覽,**無 AI 比對** |
| **iPlant**(中國) | 簡中介面,**農藥推薦不符台灣法規**(用大陸品名) |
| **ChatGPT 直接問** | 不知道台灣病蟲害 + 不知道台灣許可農藥 + 不知道農改場聯絡 |

**Gap 結構性**:GPT-4o Vision 2024 才達到 70%+ zero-shot 辨識率;台灣農業試驗所(TARI)有完整圖鑑但未商業化;台灣許可農藥清單 + 採收安全期是強烈的在地化護城河。

## cropscan 在做什麼

```
農民輸入:作物 + 病徵描述(代替真實照片,prototype 階段)
        │
        ▼
  ┌──────────────────────────────────┐
  │ pest_db.py 純資料 corpus(25+ 種台灣本土病蟲害) │
  │  - 每筆: crop / disease / 症狀關鍵字 / 嚴重度 /  │
  │    台灣許可農藥 / 採收安全期 / 預防 / 何時聯絡農改場 │
  └────────────────┬─────────────────┘
                   │
                   ▼
  ┌──────────────────────────────────┐
  │ Claude 比對症狀 vs corpus           │
  │  → Top 3 候選 + confidence 0-100   │
  │  → 命中症狀關鍵字明細               │
  │  → 嚴重度推估(輕 / 中 / 重)        │
  │  → 不確定時建議聯絡農改場           │
  └────────────────┬─────────────────┘
                   │
                   ▼
  ┌──────────────────────────────────┐
  │ 純函式渲染診斷報告                  │
  │  - Top 3 病害 + confidence + 症狀  │
  │  - 嚴重度判斷標準(corpus 給的)     │
  │  - 台灣許可農藥列表(corpus 純函式)│
  │  - 採收安全期 + 預防建議            │
  │  - 區域農改場聯絡電話               │
  └────────────────┬─────────────────┘
                   │
                   ▼
        診斷報告 markdown
        (老闆 / 農改場推廣員審閱)
```

3 個關鍵架構決策:

1. **`pest_db.py` 純資料 corpus**(25+ 病蟲害):每筆完整 metadata(典型症狀、嚴重度、台灣許可農藥、安全期、預防、何時必聯絡農改場)— 海外工具不可能複製這份在地化資料庫。
2. **農藥推薦純函式從 corpus**:LLM **絕不能編造農藥名**(避免推薦過期/違法/國外藥劑)。所有用藥建議來自 corpus 預先整理的台灣許可清單。
3. **AI 嚴格只給 confidence 不下定論**:report 用「最可能 / 疑似」措辭;low confidence 自動建議聯絡農改場;確診後仍需農民決策。

## 動作

### 純函式 keyword 比對(免 API key)

```bash
python3 cropscan.py samples/sample_input.json --no-ai
```

`samples/sample_input.json` 是南部牛番茄黃化捲葉病情境。No-AI 模式做 substring match,信心度較低(本範例 48%)但仍可看到診斷流程。

### 完整 AI 模式(需 API key)

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python3 cropscan.py samples/sample_input.json --out diagnosis.md
```

AI 做語意比對(理解「葉子顏色變黃」≈「葉片黃化」),信心度大幅提升(本範例會給 90+ TYLCV)。

### 預先產出的 demo

`examples/sample_output.md` 是 AI 模式預期結果:**Top 1 = 番茄黃化捲葉病毒病 TYLCV 92% 信心**(命中 5 個症狀:葉緣捲曲 / 黃化 / 植株矮化 / 頂芽叢生 / 果實變小 / 銀葉粉蝨)+ 嚴重度推估「中」+ 防治建議含「TY 抗病品種 (TY-Carmen / TY-Maravel)」+ 重要警告「TYLCV 病毒病無法治癒,鄰田已整批拔除是強烈警示」+ 台南農改場聯絡電話。

`examples/sample_output_no_ai.md` 是 純函式 keyword 比對的 baseline,展示無 AI 時的能力下限。

## 真正產品要有但 prototype 沒做

- **真正的 GPT-4o Vision API 接照片輸入**:目前 prototype 用文字描述代替,真實產品要接受 LINE 圖片上傳 → vision API zero-shot 辨識
- **接 TARI 完整圖鑑**:從現有 25 條擴充至 200+ 病蟲害(全台灣作物 80%+ 覆蓋)
- **農藥殘留期動態查詢**:對接農藥資訊系統,確保推薦的農藥仍在許可期內
- **多病害共感辨識**:1 株可能同時有 2 種病害,需 multi-label classification
- **歷史 confidence tracking**:用戶 feedback「對 / 不對」→ 累積準確度資料
- **LINE Bot 整合**:農民拍照上傳 → Bot 自動回診斷,不需要打開網頁
- **氣象資料整合**:雨季 / 颱風 / 高溫 等氣象條件影響病害判斷,corpus 給的 prevention_tips 可動態調整
- **農改場排程整合**:嚴重 case 直接幫農民預約農改場推廣員到府

## 商業模式

| 方案 | 月費 | 對象 |
|------|------|------|
| **Free** | 月 10 次辨識 | 試用 |
| **農民 Pro** | NT$299 / 月 | 個別農民,無限辨識 |
| **農會 / 農資行** | NT$5,000 / 月 | 白標服務,服務轄區農民 |
| **農改場授權** | NT$15,000 / 月 | 各區農改場輔助工具 |
| **農藥行 sponsored** | NT$5 / click | 推薦對應農藥的點擊抽佣 |

### WTP

- 1 次農藥誤噴損失 NT$3,000-8,000 vs Pro NT$299/月 = **單次避免誤噴即年費回本**
- 農會 / 農資行的 NT$5,000 月費白標服務 = **跟客戶綁定 + 推銷自家農藥**雙重價值

### TAM

- 全台 77 萬農家,中小型 1-5 公頃約 30 萬戶
- 取 3% 滲透率 = 9,000 戶 × NT$299 = **月 NT$270 萬 MRR / 年 NT$3,240 萬 ARR**
- 加農會 / 農改場 B2B + 農藥行 sponsored → 翻倍

## 風險評估

| 風險 | 評估 | 緩解 |
|------|------|------|
| **農委會自己推官方 AI** | 中~高 | 政府開發慢,且農委會無 SaaS 經驗;搶占心占率 |
| **AI 誤判導致農民噴錯藥** | 高 | system prompt 嚴格只用 corpus + low confidence 自動轉介農改場 + 報告免責 |
| **Plantix 等推繁中版** | 中 | 海外公司不會做台灣特有作物 corpus + 台灣許可農藥;在地化護城河深 |
| **農藥廠商法律問題** | 中 | corpus 只列許可農藥(來自防檢局公開資料);不掛特定品牌商業擔保 |
| **農民 IT 接受度低** | 中 | LINE Bot 介面(農民已用 LINE)+ 簡化操作 + 鄉鎮農會推廣員協助 |
| **季節性需求**(病害發生不平均) | 中 | 多作物 + 多季節 corpus 平衡;旺季 Pro 用戶留存到淡季 |
