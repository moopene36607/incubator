# settlekit (정산킷)

**한국 1인 크리에이터 협찬 합의서 + 정산서 AI 자동 생성기.**
**韓國 1人 創作者 品牌合作合約 + 정산서 AI 自動生成。**

브랜드 협찬 deal 정보를 입력하면, 30 초 내에:
1. **KFTC 광고 표시 의무 + 표준 약관 반영** 한 협찬 합의서 (Claude 생성)
2. **사업소득 3.3% 원천징수 자동 계산** 된 정산서 (순수 함수)

두 markdown 문서를 한 번에 출력한다.

---

## 痛點

한국 1인 크리에이터 (YouTuber, Instagram 인플루언서, Threads 계정 운영자) 가 매월 마주하는 가장 큰 행정 부담은 **세금이 아니라 협찬 deal 의 합의서·정산서 작성** 이다.

> **"1인 창작자 56%, 불공정 계약… 서울시 '유튜버 표준계약서' 만든다"**
> — 한국일보, 2022.10.23 (이 수치는 2025 년 정책 토론에서도 인용되고 있음)

> **"한국 4,000+ 명의 高수입 YouTuber 가 국세청 세무조사 명단에 포함, 협찬 광고비 신고 누락이 주요 적발 사유"**
> — Net Influencer, 2025

구체적인 페인 포인트:

- 협찬료 ₩300만 ~ ₩2,000만 받는 mid-tier 창작자가 대부분 합의서 없이 카톡으로 deal 진행 → 광고주가 게시 후 광고비 미입금하거나 추가 수정 요구하는 사고 빈발
- 합의서를 작성한다 하더라도 KFTC 「추천·보증 등에 관한 표시·광고 심사지침」 의 광고 고지 의무 (#광고, "유료 광고 포함" 표기) 를 정확히 반영하지 않아 행정조치·과태료 사고 발생
- 사업자 미등록 프리랜서의 경우 사업소득 3.3% 원천징수 + 종합소득세 신고 메커니즘을 본인이 모르고, 광고주도 어떻게 정산해줘야 할지 몰라 양쪽 다 헷갈림
- 변호사·세무사에게 합의서 1 건 검토 의뢰 시 ₩300,000 ~ ₩500,000 — 협찬료가 작은 deal 에는 ROI 가 안 맞음

## 競品 gap

| 기존 도구 | 왜 1인 창작자에게 부족한가 |
|----------|-------------------------|
| **삼쩜삼 / 자비스앤빌런즈** | 종합소득세 신고만 처리. 협찬 합의서 / 정산서는 다루지 않음 |
| **Modusign** | 기업급 전자계약 플랫폼, 월 ₩50,000 ~ ₩200,000, 1인 창작자 가격대 아님 |
| **BHSN 앨리비** | 기업 CLM (Contract Lifecycle Management), 인하우스 법무팀용 |
| **DocuSign / HelloSign** | 일반 전자서명, 한국 KFTC 광고 고지 의무 모름 |
| **Notion / Google Docs 템플릿** | 매번 수동 복붙, KFTC 조항 변경 시 미반영, 정산 계산 별도 |
| **MCN 사내 시스템** | MCN 소속 크리에이터만 접근 가능, 1인 독립 활동자는 사용 불가 |

**갭**: "광고주 + 광고비 + 콘텐츠 종류" 만 입력하면 KFTC 합의서 + 사업소득 정산서를 한 번에 출력하는 SaaS 가 검색 결과에서 확인되지 않음. 이게 settlekit 이 채우려는 공간.

## 시장 규모 (2025 년 기준)

- **타겟 창작자 수**: 한국 활성 1인 크리에이터 50,000 ~ 100,000 명 (sub 1 만 ~ 50 만 mid-tier 기준)
- **연간 협찬 deal 수**: 인당 평균 12 ~ 36 건 → 시장 전체로 60 만 ~ 360 만 건 / 년
- **현행 1 건당 합의서 + 정산서 처리 시간**: 평균 1.5 ~ 3 시간 (직접 작성) 또는 ₩300,000+ (외주)
- **시장 잠재 규모**: 활성 사용자 30,000 명 × 월 ₩9,900 = 월 약 ₩3 억 ARR (10 ~ 15% 침투율 가정)

## settlekit 이 어떻게 작동하는가

```
협찬 deal JSON
(창작자 / 광고주 / deal 조건 / 인도 콘텐츠 목록)
       │
       ▼
   ┌───────────────┬────────────────┐
   ▼               ▼
순수 함수 계산  Claude API 생성
(settlement.py)  (KFTC 시스템 프롬프트)
   ▼               ▼
정산서 markdown  합의서 markdown
   │               │
   └───────┬───────┘
           ▼
      한 번에 두 파일 출력
```

**금액 계산은 절대 LLM 에 맡기지 않는다.** 환각 risk 가 너무 크기 때문에 모든 세금·실수령액 계산은 `settlement.py` 의 순수 함수가 담당하고, LLM 은 합의서 본문만 생성한다.

### 합의서가 보장하는 KFTC 준수 사항

자동 생성되는 합의서에는 다음 KFTC 필수 조항이 항상 포함된다:

- **제4조 표시·광고 의무**: YouTube 영상 5초 이내 "유료 광고 포함" 자막, Instagram 첫 줄 #광고 해시태그
- **표시 누락 시 책임 분배**: 행정조치·과태료 발생 시 직접 원인 제공자 부담
- **콘텐츠 활용 범위 + 기간**: 「갑」 의 2차 활용 (자사 SNS / 유료 광고 매체) 한도 명시
- **동종 카테고리 비경쟁 의무 + 위약금**

### 정산서가 보장하는 세무 정확성

- 사업소득 (3.3%) 또는 일시적 사업소득 (22%) 자동 분기
- 사업자등록 여부에 따른 부가세 (10%) 처리 분기
- MCN/플랫폼 수수료 차감
- 종합소득세 신고 시 그대로 활용 가능한 형식

## 동작시키기

### AI 끄고 정산서만 (API key 불필요)

```bash
python3 settlekit.py samples/sample_input.json --no-ai --out-dir output/
```

`output/settlement.md` 가 생성됨. 합의서는 LLM 호출이 필요하므로 이 모드에서는 생략됨.

### 풀 모드 (합의서 + 정산서)

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python3 settlekit.py samples/sample_input.json --out-dir output/
```

`output/contract.md` (합의서) + `output/settlement.md` (정산서) 두 파일을 출력. API 호출 비용은 1 deal 당 약 ¢2.

### 사전 생성된 데모

`examples/contract.md`, `examples/settlement.md` 에 미리 생성된 결과물이 있어 API key 없이도 결과를 볼 수 있다. Sample 시나리오:
- 창작자: 김미나 (78K subs YouTube + 32K Instagram)
- 광고주: ㈜ 글로우랩 (비건 콜라겐 세럼)
- 광고비: ₩5,000,000
- 콘텐츠: YouTube 비디오 1편 + Instagram 릴스 3편 + 스토리 2회
- 정산 결과: gross ₩5,000,000 → 원천징수 -₩165,000 → 실수령 **₩4,835,000**

## 검증된 smoke test

- ✅ 표준 사업소득 3.3%: ₩5,000,000 → 원천징수 ₩165,000, 실수령 ₩4,835,000
- ✅ 사업자 등록 + 부가세 10%: ₩5,000,000 + VAT ₩500,000 - 원천징수 ₩165,000 = 실수령 ₩5,335,000
- ✅ 일시적 사업소득 22%: ₩500,000 → 원천징수 ₩110,000
- ✅ MCN 수수료 차감: ₩5,000,000 - ₩165,000 - 수수료 ₩1,000,000 = ₩3,835,000

## 프로젝트 구조

```
settlekit/
├── README.md
├── settlekit.py          # CLI 메인 (LLM 합의서 생성 + 정산서 호출)
├── settlement.py         # 순수 함수 정산 계산
├── samples/
│   └── sample_input.json # K-beauty 협찬 시나리오 (78K YouTuber + GlowLab)
├── examples/
│   ├── contract.md       # 사전 생성 합의서 (10 조항, KFTC 준수)
│   └── settlement.md     # 사전 생성 정산서 (3.3% 자동 적용)
└── requirements.txt
```

총 약 350 줄 Python. 의존성은 `anthropic` (AI 합의서 생성 시).

## 진짜 제품에 필요하지만 prototype 에는 없는 것

- **전자서명 통합**: 양 당사자 클릭 한 번으로 합의서 서명 (Modusign / NaverSignVerification API 연동)
- **국세청 홈택스 연동**: 사업자등록번호 자동 검증, 원천징수 자료 자동 신고
- **PDF 출력 + 도장 이미지**: 한국 비즈니스 관행상 PDF + 도장 필수
- **Deal 이력 데이터베이스**: 같은 광고주와 반복 deal 시 합의서 자동 재사용
- **종합소득세 신고용 1 년치 통합 리포트**: 매년 5 월 신고 시즌 핵심 기능
- **KFTC 가이드라인 자동 업데이트**: 공정위 고시 RSS 모니터링, 규정 변경 시 합의서 템플릿 자동 갱신
- **다국어 협찬 (한일 / 한중 cross-border)**: 한국 K-content 일본 진출 시 양국 법규 준수 합의서 (kosmelingo 와 시너지)
- **Stripe / 토스페이먼츠 지급 자동화**: 합의서 서명 → 인도 확인 → 자동 입금까지 한 흐름

## 비즈니스 모델

| 플랜 | 가격 | 대상 |
|------|------|------|
| **Free** | 월 3 deal 까지 | 신규 창작자 |
| **Solo** | **₩9,900/월** (월 deal 무제한) | mid-tier 1만 ~ 50만 subs 창작자 |
| **Pro** | **₩19,900/월** (Solo + 전자서명 + 1 년치 종소세 리포트) | 高수입 200만+ 비즈니스 모드 창작자 |
| **MCN/에이전시** | **₩99,000/월** (창작자 5 명) | 中小 MCN, 매니저 운영 |
| **Enterprise** | 별도 견적 | 50+ 창작자 풀 운영 MCN |

**WTP 근거**:
- 변호사 합의서 검토 1 건 ₩300,000 → Solo 30 개월치
- 세무사 종소세 신고 보조 ₩200,000+ → Pro 10 개월치
- 협찬료 미수금 사고 1 건 ₩2,000,000+ → Solo 17 년치

## 초기 distribution

1. **Blind 한국판 광고·마케팅·미디어 카테고리** — "협찬 합의서 안 받았다가 ₩300만 떼였어요" 류 대화에 직접 솔루션 답변
2. **유튜버 카페 (네이버 카페)** — 「유튜버스타」, 「인플루언서 마인드」 등 30만+ 멤버 커뮤니티
3. **MCN 매니저 LinkedIn outreach** — 100 ~ 500 명 풀 관리하는 중소 MCN 에 White-label 제안
4. **이도배 (이지웅) 등 크리에이터 운영 콘텐츠 채널 협찬** — 그 자체가 데모 영상이 됨
5. **공정위 / 한국언론진흥재단 1인 미디어 교육 프로그램 partner** — 신규 창작자 onboarding 채널

**1차 목표**: 첫 100 명 paying Solo 사용자 = 월 약 ₩100 만 MRR. 3 ~ 6 개월 내 달성하면 PMF 시그널 충족.

## 리스크 평가

| 리스크 | 평가 | 완화책 |
|--------|------|--------|
| **공정위 / 정부 자체 도구 출시** | 낮 | 정부 도구는 보통 가이드 PDF 수준에서 멈춤, SaaS 까지 만들지 않음 |
| **삼쩜삼 등이 협찬 정산 기능 확장** | 중 | 종소세 위주 회사, 광고 합의서는 법무 영역으로 진입 부담 큼 |
| **YouTube / Instagram 자체 합의서 도구 빌트인** | 중 | 글로벌 플랫폼이라 한국 KFTC 정확 반영 가능성 낮음 |
| **합의서 환각으로 인한 법적 책임** | 中~高 | 모든 합의서에 "법무 검토 필요" 면책 + 보수적 기본값 + 위험 조항은 [별도 협의] 표기 |
| **변호사 직역 침해 시비** | 중 | "초안 자동화 도구" 포지셔닝, 검토는 사용자가 변호사에게 의뢰 권고 |
| **KFTC 가이드라인 개정** | 중~高 | 정기 모니터링 + 즉시 템플릿 업데이트, 이는 오히려 lock-in 효과 |
