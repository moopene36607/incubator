# carlens(車透鏡)

**台灣中古車多源資料一致性檢查 — 把照片、行照、賣方宣稱、配備清單交叉比對,找出**詐騙紅旗**(調表 / 事故 / 泡水 / 拼裝)。

純函式跑 7 個維度的「跨來源一致性檢查」(內裝磨耗 vs 行照里程 / 引擎室油漆 vs 無事故宣稱 / 底盤塗裝 vs 無泡水宣稱 / 配備版本 vs 年式 / 車身漆面 / 輪胎磨耗 / 車身號碼)→ 純函式累加 risk_score 0-100 → LLM 寫故事化解釋 + 開價合理性 + 議價建議 + 盡職調查清單。

把「**這台 685K 看起來很漂亮的 18 年 4 萬公里 Camry**」變成「**4 條獨立證據暗示是 8-10 萬公里 + 有事故 + 疑似泡水拼裝車,合理價 NT$38-45 萬**」。

---

## 痛點

台灣每年中古車交易約 **100 萬輛**,業內估**詐騙車 3-5%**(調表 / 事故重組 / 泡水 / 拼裝 / 行照偽造 / 贓車)。一般買家完全沒能力辨識:

> **「8891 看到一台 18 年 Camry 4 萬公里只要 68 萬,興沖沖跑去看,簽完約一個月後變速箱壞掉去原廠檢查,讀 ECU 才知道實際里程已 9.8 萬,根本是調表車。」** — Mobile01 中古車版真實案例

> **「徵信社查車況一台 NT$3-5K,跟我朋友 4 個都查了 = 1.5-2 萬,光看車就花這筆,還沒砸下去。」** — Dcard 中古車板

> **「8891 / 比比看 listing 只有照片,沒有任何 AI 風險評估,我自己看不出來哪裡可疑。」** — PTT car 板

具體痛點:

- **調表車**:電子里程錶大部分可調(賣方店家工具普及),調表後一般買家無法察覺
- **事故車**:引擎室 / 車身重噴 / 板金修補後一般人看不出來
- **泡水車**:底盤重新塗裝掩蓋,水痕短期看不到
- **拼裝車**:車身號碼篡改,贓車或多車殘骸組合
- **行照偽造**:年式 / 里程 / 顏色都可改
- **徵信社 NT$3-5K/件車況檢查太貴**,一個買家通常看 3-5 台車 = NT$15K-25K 預算花在「還沒買車前」
- **8891 / 比比看 / Yahoo 中古車**只有 listing,**沒人做 AI 風險評估**
- **ChatGPT 直接問**:不能看照片(無 vision)+ 不知道台灣中古車市場
- 政府監理站「車籍查詢」只能查官方紀錄,**詐騙車的紀錄通常被洗掉了**

## 為什麼現在沒有對的工具

| 既有工具 | 為什麼不行 |
|----------|-----------|
| **徵信社車況檢查** | 一案 NT$3-5K;耗時 1-2 週;一個買家看 3-5 台車成本 NT$15-25K |
| **原廠車況檢查** | 一台 NT$3-5K + 預約等 1-2 週;**買家還沒簽約賣方多半不准進原廠** |
| **8891 / 比比看 / 中古車網站** | 只有 listing 照片;**完全沒做 AI 風險評估** |
| **ChatGPT / Claude 直接問** | 不能看照片;不知台灣中古車市場行情 |
| **監理站「車籍查詢」** | 只查官方紀錄;**詐騙車的紀錄通常已洗掉** |
| **車聯網 CARFAX(海外)** | 完全不認識台灣車輛;中文使用者用不了 |
| **自己看 + Google** | 一般人完全沒專業背景看不出引擎室 / 底盤 / 漆面差異 |

**Gap 結構性**:中古車詐騙痛點明確 + 公開資料(照片 + 行照 + 配備)足夠做交叉比對 + AI vision + 知識庫成熟,**但沒人整合**。Google「台灣 中古車 AI 風險評估」零中文 SaaS 結果。

  │   - 引擎室狀態(原廠未動 / 局部重噴 / 全面重噴 / 改裝痕跡)  │

  └────────────────┬─────────────────────────────────────────┘

  ┌──────────────────────────────────────────────────────────┐

  │   2. 年式 vs 配備版本(CarPlay 2018+ / 盲點偵測 2016+)      │
  │   3. 引擎室油漆 vs 無事故宣稱(引擎室重噴 = 前方碰撞)         │
  │   4. 底盤生鏽 / 新塗裝 vs 無泡水宣稱                         │
  │   5. 車身漆面一致性(局部色差 / 重噴痕跡 = 板金)              │
  │   6. 輪胎磨耗 vs 行照里程(輪胎 6-8 萬 km 換一次)             │
  │   7. 車身號碼真偽(可疑 / 缺失 → 強制 CRITICAL)              │
  └────────────────┬─────────────────────────────────────────┘
                   │
                   ▼

  │ 純函式 risk_score 累加 + 切檔                              │
  │   LOW ≤ 20 / MEDIUM ≤ 45 / HIGH ≤ 75 / CRITICAL > 75       │
  │ critical-severity 訊號(VIN / 極端調表)強制升 CRITICAL       │
  └────────────────┬─────────────────────────────────────────┘
                   │
                   ▼
  ┌──────────────────────────────────────────────────────────┐
  │ Claude 寫:                                                │
  │   - 故事化解釋(把多個訊號串成「這台車真實歷史可能是 X」)    │
  │   - 開價合理性(乾淨同款車 vs 風險調整後 vs 賣方開價)        │
  │   - 最終建議三選一(不買 / 議價條件式買 / 鑑定後買)           │
  │   - 進一步盡職調查清單(原廠檢查 / VIN 鑑識 / 司法院車籍...) │

3 個關鍵架構決策:

1. **多源 fusion 是核心價值**:單看任何一張照片難以判斷,但「**內裝磨耗 6/10 + 行照 38K km + 引擎室重噴 + 底盤新塗裝**」4 個獨立來源組合 → 故事一目了然。這就是 multi-modal fusion 的關鍵 — **單模態看不出問題,組合看就明顯**。
2. **數字 100% 純函式**:7 個維度檢查邏輯 + risk_score 累加 + 等級切檔全在 `inspector.py`;LLM 永不算數字。
3. **故事化解釋 + 開價對照**:LLM 角色不是「找問題」(純函式做),而是「**把問題串成故事 + 算合理價**」 — 賣方開價 NT$685K vs 乾淨同款 NT$650K vs 實際應為 NT$380-450K。

## 動作

### 純函式檢查(免 API key)

```bash
python3 carlens.py samples/car_profile.json --no-ai --out output.md
```

`samples/car_profile.json` 是 Toyota Camry 2018 / 4 萬 km / 685K — 賣方宣稱無事故無泡水,但實際多項證據矛盾。

輸出含 7 維檢查結果 + 4 條風險訊號 + score 70 = HIGH。

### 完整 AI 模式(需 API key)

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python3 carlens.py samples/car_profile.json --out output.md
```

加上 Claude 寫故事化解釋(把 4 條訊號串成「拼裝車」故事)+ 開價合理性(賣方 685K vs 乾淨 650K vs 實際 380-450K)+ 最終建議(不建議購買)+ 8 條盡職調查清單。

### 預先產出的 demo

`examples/sample_output.md` AI 模式完整報告,展示:
- 多源 fusion 揭露「調表 + 事故 + 泡水 + 板金」故事
- 開價超出實際合理價 NT$235-305K(50-80% 溢價)
- 最終建議**不購買**(替代條件 NT$380K + 違約保障條款)
- 8 條盡職調查 — 含**ECU 讀取最大記錄里程**(賣方無法竄改)

`examples/sample_output_no_ai.md` 純函式模式輸出。

## 真正產品要有但 prototype 沒做

- **真實 Vision API 整合**:用戶上傳多張車輛照片 → Claude Vision API 自動抽結構化特徵(prototype 用文字描述代替)
- **8891 / 比比看 listing 自動拉取**:用戶貼 listing URL → carlens 自動抓所有照片 + listing 資訊
- **市場行情即時對照**:從 8891 抓同款車近 30 天成交紀錄,給出實時市場中位數
- **VIN 監理站 API**:接政府公開車籍資料(年式 / 引擎號 / 車身碼 / 違規紀錄)
- **保險理賠紀錄查詢**:接財團法人保險事業發展中心(用戶授權 + 賣方授權)
- **車牌 / 照片倒查**:用 image hash 在 Google / 8891 過去 listing 倒查(常見「事故重組車多次掛賣不同價格」)
- **LINE Bot**:LINE 上傳照片 + listing URL → 1 分鐘回 risk 報告
- **車商白名單 / 黑名單**:眾包車商評價(隱私 / 法律需謹慎)

### WTP 計算

- 徵信社車況檢查 NT$3-5K vs carlens NT$299 = **10-15x 便宜**
- 一台調表車「便宜買到貴 NT$10-20 萬」→ NT$299 預防 = **300-700x 回本**
- 中古車商每月看 30-50 台車進貨,**Pro NT$799/月** vs 自己花 NT$3K × 5 鑑定 = **18x 便宜**

### TAM

- 台灣每年中古車交易 100 萬輛 + 一般買家通常看 3-5 台 = **300-500 萬次 lookup**
- 取 1% 滲透 = 3-5 萬人 × NT$299 = **年 NT$900-1,500 萬 ARR (Pay-per-use)**
- 加 Pro + Studio 車商 + B2B 保險 / 銀行 → 年 ARR **NT$3,000-5,000 萬**
- 横移日韓港新 → 翻倍

## 風險評估

| 風險 | 評估 | 緩解 |
|------|------|------|
| **誤判導致買家投訴賣方 / 名譽損害** | 高 | UI / report 多次強調「初步參考」+「請洽原廠 / 公正第三方鑑定」+ 報告不主動傳給賣方 |
| **Vision 模型誤判** | 高 | 報告底部標示「依用戶上傳照片」+ 鼓勵多角度照片;Pro 版加 human-in-loop 第二次審核 |
| **賣方 / 中古車公會 反感** | 中 | 走 partner 路線(8891 / 比比看 integration),長期提升市場透明度反而利好誠實車商 |
| **法律風險(被指控誹謗 / 妨害名譽)** | 高 | UI 明確「**僅依用戶上傳資料計算**」+「**最終購買決策在用戶**」+ 不公開特定車輛 risk score |
| **個資 / 車輛資料隱私** | 高 | stateless API call;報告不上雲;企業版 self-host LLM |
| **政府監理站不開放 API** | 中 | 與政府協商;最壞情況純依用戶上傳資料分析 |
| **詐騙手段升級(新型 anti-detection)** | 中 | 持續更新 inspector.py 維度;眾包受害者經驗 |
