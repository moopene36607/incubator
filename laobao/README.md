# laobao(勞報)

**台灣 SOHO 接案族 / 中小企業專用 — 勞報單 + 二代健保補充保費自動算 + 文件草稿產生。**

輸入金額和情境,30 秒內知道:
- 自己實領多少
- 公司要代扣多少
- 一張可以直接給對方公司的勞報單草稿

---

## 痛點

台灣 SOHO 接案的人最頭痛的不是工作,是收款後的稅務:

> 「公司請我接案,我不知道要不要被扣 10%?」
> 「補充保費 NT$2 萬以上才扣?還是 NT$2 萬 10 元?」
> 「我這次的收入是 9A 執行業務還是 50 薪資所得?差很多耶」
> 「公司說『沒辦法報勞報單,要不你開發票』,我又沒有公司⋯」

實際論壇證據:

- **PTT soho 版**:「一直都不太清楚二代健保的問題」「我都狂被扣」
- **Medium 接案彙整文**:「合作廠商還是需要發票公司沒辦法報勞報的問題,**有時會因此失去這個客戶**」
- **swap.work 自己的 blog 承認**:「許多公司會因為這些繁雜的手續就不想用勞報單來核銷」

每年費率還會調整(2025 年二代健保 2.11%,門檻 NT$20,000;所得稅扣繳起扣 NT$20,010),自製 Excel 一更新就壞掉。

## 為什麼現在沒有對的工具

| 現有方案 | 為什麼不夠 |
|----------|------------|
| **swap.work** | 必須透過該平台收款才能用,收 9% 平台費。等於強迫綁定金流。 |
| **健保署官網試算工具** | 只算單筆數字,不產文件、不記錄歷史、不解釋類別。 |
| **freee TW / 雲端發票工具** | 為公司端設計,SOHO 接案者用不到。 |
| **自製 Excel** | 費率一變就過期。新手連 9A / 9B / 50 都分不清。 |

缺口很明確:**純工具型、不綁金流、月費 NT$99 以下、含 AI 智能分類、可下載勞報單**。這是這個 prototype 想填的。

## laobao 做什麼

```
{ 金額 + 情境描述 + 雙方資料 }
            │
            ▼
   tax_rules.py 純函式計算
   ( + Claude 自動判斷所得類別,情境模糊時用 )
            │
            ▼
   勞報單草稿 (markdown / 印表機格式)
   + 拆解明細 + 法規說明
```

三個關鍵元件:

1. **`tax_rules.py`** — 純 Python 計算函式,所有費率/門檻常數寫在最上方,每年調整一次就好。`calculate()` 回傳 `CalcResult`,內含實領/扣繳/說明。
2. **AI 自動分類**(`income_type: "auto"`)— 使用者輸入自然語言情境(「我兼職在補習班教國中數學」),Claude 判斷屬於 9A / 9B / 50。對 SOHO 新手是 killer feature。
3. **勞報單草稿輸出** — 直接產生印表機友善的勞報單版式,使用者印出 → 雙方簽名 → 公司報帳。

## 試跑(無 API key 也可)

`income_type` 直接指定時,不會打 API:

```bash
python3 laobao.py samples/sample_input.json
```

預先產出的範例在 `examples/sample_output.md`,不跑也能看。

## 試跑 AI 自動分類(需 API key)

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python3 laobao.py samples/sample_input_auto.json
```

第二份 sample 故意填 `income_type: "auto"` + 情境「兼職在補習班教國中數學」,LLM 應該判斷為 50(薪資所得,雇傭關係),而不是 9A。

## 專案結構

```
laobao/
├── README.md
├── laobao.py              # CLI 主程式
├── tax_rules.py           # 純函式計算邏輯(費率常數寫在頂部)
├── samples/
│   ├── sample_input.json       # 設計接案 NT$30,000,9A
│   └── sample_input_auto.json  # 補習班鐘點費 NT$18,000,LLM 判斷
├── examples/
│   └── sample_output.md   # 預先跑好的輸出
└── requirements.txt
```

總共約 350 行 Python,沒有額外依賴(只有 `anthropic`,且只在 LLM 模式才需要)。

## 已驗證的計算正確性(smoke test)

- NT$15,000 9A:免扣稅、免補充保費 ✅
- NT$20,005 9A:免扣稅(未達 NT$20,010),扣補充保費 NT$422 ✅
- NT$30,000 9A 工會投保者:扣稅 NT$3,000、不扣補充保費 ✅
- NT$50,000 9B 已累計 NT$150,000:剩餘免稅額 NT$30,000 邏輯正確 ✅

## 沒做但真實產品要有的(下一步)

- **PDF / 圖片輸出** — 目前只到 markdown,真正寄給公司還是要 PDF。`reportlab` 或 `weasyprint`。
- **勞報單歷史記錄 + 年度總覽** — SOHO 要報稅時最痛的是「整年到底領了多少」。
- **發票替代邏輯** — 部分公司只接受發票,提供「找代開發票合作社 / 中小企業協會」對接。
- **二代健保補充保費月份累計判斷** — 補充保費另一條規則是同一公司同一個月累計達當月最低工資 4 倍才扣。本 prototype 簡化掉。
- **每年費率自動更新** — 排程抓健保署 RSS / 財政部公告,費率變了主動通知付費客戶。
- **企業版多人帳號** — 5–30 人公司的會計兼總務想批次處理。
- **行動版 PWA** — 接案者很多在外面,手機算最方便。

## 商業模式

| 方案 | 價格 | 對象 |
|------|------|------|
| Free | 每月 3 張 | 試用 SOHO |
| Pro | NT$99 / 月 | 個人 SOHO,無限張、含 AI 分類、PDF 下載 |
| Team | NT$499 / 月 | 5 人以下小公司,共用模板、批次處理 |
| Pay-as-you-go | NT$15 / 張 | 偶爾才用的人 |

距離 PMF 的關鍵是:**讓使用者第一次來就 30 秒看到答案**,不需要註冊、不需要刷卡。免費版做到這件事,Pro 版用 PDF 下載 + 歷史記錄變現。

## 早期 distribution 渠道

1. **PTT soho 板** — 在「二代健保」「勞報單」相關討論串底下回覆「我做了個工具,30 秒算給你看」
2. **Dcard 工作版 / Soho版**
3. **Threads / X.com 上活躍的台灣 freelancer KOL**(像「接案這檔事」、「設計師接案地獄」這類粉專)
4. **本來付 swap.work 9% 平台費的人** — Google Ads 鎖定 swap.work 相關關鍵字
5. **小規模設計工作室、前端/後端外包** — 直接寄信給有公開接案網站的工作室

第一個目標:**前 100 個付費 Pro 用戶 = NT$9,900 MRR**,3 個月內達成代表 PMF 訊號到位。

## 風險

- **政府工具改版**:健保署若推出含文件下載的免費試算工具,影響低端市場(機率中,但通常政府進度慢)
- **freee 台灣等大公司下游延伸**:他們現在主打公司端,若延伸到 SOHO 個人,本工具需要快速建立社群護城河
- **一次性購買心理**:SOHO 可能買單一次後不續訂。對策:每年費率調整時自動寄信「今年規則改了,你的歷史記錄已自動更新」
