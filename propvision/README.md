# propvision(房視)

**台灣房屋室內照片 Vision AI 估價 — 上傳照片 + 物件資料,30 秒拿到「估值區間 + 折舊明細 + 加減分透明」報告。**

把「房仲報價是否被裝潢矇」「自售開價要不要砍 15%」這些每位看房 / 賣屋族都在猜的問題,壓成一份可追溯的數據報告。

---

## 痛點

台灣每年 30 萬棟交易 + 60,000 位房仲業務員,但**估價工具完全不看屋況**。

> **「仲介帶看說老屋但裝潢還新,報價比周邊高 15%,後來查實價發現完全灌水。」**
> — Dcard 房地產版 2024/03

> **「自售要開價多少才合理?591 行情看不出我家裝潢加分多少。」**
> — PTT home-sale 板常見

具體痛點:

- **591 / 樂居 / 信義 AI 估價** 只看交易歷史**完全不看照片**,新裝潢 / 漏水 / 學區都不計入
- 同棟同戶型可能因裝修差距 ±20% 成交價,但**現有工具給的是「均值」而非「我家的估值」**
- 房仲估價 / 銀行貸款估價 主觀,屋主沒有客觀工具 cross-check
- 看房買方無法快速判斷「這間值不值報價」,**每場看房後都要花 1-2 小時自己算**

## 為什麼現在沒有對的工具

| 既有工具 | 為什麼不行 |
|----------|-----------|
| **591 / 樂居 / 信義 AI 估價** | 純交易歷史數字,**完全不看照片屋況** |
| **內政部實價登錄** | 只給原始成交資料,**無估價邏輯** |
| **Opendoor / HouseCanary** (美) | 不在台灣,iBuying 商業模式不同 |
| **房仲估價 / 銀行貸款估價** | 主觀,沒有透明明細 |
| **ChatGPT 直接問** | 不知道你家照片 / 屋況 / 學區 / 交通 |

**Gap 結構性**:實價登錄 open data 公開可用(內政部 2021 起完整樓層資料),但**沒有任何服務把照片屋況與成交價格連結**。GPT-4o Vision 2024 才達到可實用門檻 — 這是 *新可能*。

## propvision 在做什麼

```
物件資料 + 室內照片(3-5 張)
        │
        ▼
  ┌──────────────────────────────────┐
  │ Claude Vision 解析照片             │
  │  - 抽 renovation_score 1-10        │
  │  - 抽 renovation_notes(具體觀察)  │
  │  - 抽 concerns(漏水 / 壁癌等)      │
  │  - 不過度樂觀,有瑕疵直接給低分     │
  └────────────────┬─────────────────┘
                   │
                   ▼
  ┌──────────────────────────────────┐
  │ pricing_model.py 純函式估價         │
  │  1. base = 區域均價 × 坪數          │
  │  2. × age_factor (每年 -1.3%,bottom 0.55) │
  │  3. × floor_factor (1F/4F/頂樓 penalty) │
  │  4. × orientation_factor (南向最佳) │
  │  5. × renovation_factor (1-10 → 0.85-1.15)│
  │  6. × (1 + Σ adjustments capped ±15%) │
  │  7. ±8% 區間                        │
  └────────────────┬─────────────────┘
                   │
                   ▼
        估值報告 markdown
        - 自售區間 + 開價建議 + 急售底價 + 房仲收購
        - 6 步驟折舊明細(透明可追溯)
        - 加減分明細(電梯/車位/捷運/學區/嫌惡)
        - AI 寫的人性化「為什麼這個價」說明
```

3 個關鍵架構決策:

1. **金額計算 100% 純函式**(`pricing_model.py`)— LLM 永不算錢,避免幻覺。所有數字計算公式可單元測試、可重現。
2. **AI 只做語意解析**:① 照片描述 → 結構化 renovation_score ② 為估值結果寫人性化說明。**估價邏輯本身是純函式**。
3. **加減分透明列出 + 細項上限 ±15%**:電梯(+2%)、車位(+5%)、近捷運(+6%)、學區(+4%)、嫌惡設施(-5% ~ -8%) — 累計 cap 防止極端值。

## 動作

### 結構化 JSON 輸入(免 API key)

```bash
python3 propvision.py samples/sample_input.json --no-ai
```

`samples/sample_input.json` 是台北市大安區 25.6 坪 27 年屋,8F/12F、東南向、有電梯車位、近捷運 280m、明星學區、renovation_score 7。

### AI Vision 解析照片描述(需 API key)

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python3 propvision.py samples/sample_input.json --out output.md
```

AI 從 photo_descriptions 抽 renovation_score + notes + concerns,然後走純函式估價。

### 預先產出的 demo

`examples/sample_output.md` 是完整估值報告:
- **自售區間 NT$ 2,091 萬 ~ 2,455 萬**,中位 **NT$ 2,273 萬**
- 開價建議 2,387 萬 / 急售底價 2,091 萬 / 房仲收購 1,932 萬
- **6 步驟折舊明細**:Base 2,816 萬 → 屋齡 ×0.649 → 樓層 ×1.0 → 朝向 ×1.03 → 裝修 ×1.05 → 加減分 +15% = 2,273 萬
- **加減分明細**:有電梯 +2% / 車位 +5% / 近捷運 +6% / 學區 +4% = +17% 累計(cap @ +15%)

## 已驗證 smoke test

- ✅ Age factor:0 年→1.0,35 年→0.545,>40 年 cap at 0.55
- ✅ Floor factor:1F 0.97 / 4F 0.98 / 頂樓 0.97 / 9F+(中高層)1.02
- ✅ Orientation:南向 1.03、東南向 1.03、西向 0.97、北向 0.96
- ✅ Renovation:1→0.85, 5→0.969, 10→1.15
- ✅ Poor old scenario:新北 40 年 4F 北向頂樓無電梯無車位 + 嫌惡設施 × 2 → 從原始 600 萬跌到 252 萬(合理)
- ✅ Luxury new scenario:信義 50 坪 3 年 15F 南向全配 → 從 7,500 萬漲到 9,724 萬

## 真正產品要有但 prototype 沒做

- **真正的 vision API 接 GPT-4o Vision / Claude Vision**:目前 prototype 用照片描述代替,真實產品要接受 image upload
- **接內政部實價登錄 API 動態 sync**:目前 region_avg_price_per_ping 是手動輸入,真實產品自動從 plvr.land.moi.gov.tw 抓即時資料
- **行政區子分區**:大安區內仁愛 / 信義 / 敦化北路差異大,需子分區 lookup table
- **建商品牌調整**:新成屋(<5 年)有建商溢價(忠泰、皇翔等品牌價差 ±5-10%)
- **歷史成交筆數加權**:近 1 個月 5 筆成交 vs 近 6 個月 50 筆,均價可信度不同
- **戶型 / 隔局加分**:3 房 / 邊間 / 雙陽台 等隔局項
- **bank 估價對比**:銀行貸款估價(通常更保守),產出比較表給屋主議價用
- **PDF 報告 + 公司 logo**:房仲業務員 white-label,個性化品牌

## 商業模式

| 方案 | 價格 | 對象 |
|------|------|------|
| **Free** | 月 2 份估價 | 試用 |
| **B2C 個人** | **NT$1,500 / 份**(pay-per-use) | 屋主自售 / 買方盡職調查 |
| **B2B 房仲事務所** | NT$8,000 / 月(含 20 份) | 5-30 人房仲事務所 |
| **B2B 房仲業務員** | NT$3,000 / 月(個人,含 8 份) | 自由房仲業務員 |
| **Enterprise** | 客製 | 連鎖房仲(信義 / 永慶 / 中信) |

### WTP 計算

- 房仲業務員一月成交 1-2 件 × 抽佣 NT$10-30 萬 → NT$1,500/份估值報告是衝動消費
- 屋主自售開價錯 5% = 損失 NT$50K-100K(2,000 萬物件)= NT$1,500 是極好保險
- 買方盡職調查避免溢價 = 同等價值

### TAM

- 60,000 位房仲業務員 × 5% 月訂閱 = 3,000 人 × NT$3,000 = **月 NT$900 萬 MRR / 年 NT$1.08 億 ARR**
- 加 B2C pay-per-use 屋主端 30 萬交易 × 10% 滲透 × NT$1,500 = 年 NT$4,500 萬
- 加 Enterprise 連鎖房仲(信義 / 永慶) 白標 → 翻倍

## 風險評估

| 風險 | 評估 | 緩解 |
|------|------|------|
| **591 / 樂居 自己加 vision 估價** | 中~高 | 他們廣告收入結構衝突;搶心占率;持續強化「實價可追溯」護城河 |
| **AI 過度樂觀 / 悲觀 vision 評分** | 高 | system prompt 嚴禁過度樂觀 + few-shot 1-10 rubric 明確;報告寫明「僅供參考」 |
| **房地產估價師工會反彈** | 中 | 明確定位「議價起點參考」非「正式不動產鑑價」+ 免責聲明 |
| **法律 / 房地產交易糾紛** | 中 | 報告免責聲明 + 不掛保證 + 第三方檢視建議 |
| **個資 / 房屋照片隱私** | 中 | stateless API call;企業版可走 self-host LLM;不長期儲存照片 |
| **連鎖房仲(信義 / 永慶)自建** | 中 | 連鎖開發慢 + 工會內部利益;搶獨立業務員市場 |
