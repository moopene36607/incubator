# trailmatch(山路配)

**台灣登山路線 AI Personalization — 經驗 / 體能 / 天數 / 季節 / 出發地 → Top 3 個人化山岳推薦 + 裝備提示 + 安全警告。**

把「我這種程度可以爬什麼山?」從 PTT Hiking 板每週 3-5 次的求助貼文,壓成 30 秒可拿到的個人化報告。

---

## 痛點

台灣 266 座 3,000m+ 百岳 + 數百座中級山,但**選山靠經驗 + 朋友推薦 + 健行筆記滑半小時**。

> **「新手適合爬什麼山?」「體力普通可以挑戰南湖嗎?」「第一次百岳從哪座開始?」**
> — PTT Hiking 板每週 3-5 次重複求助

具體痛點:

- **新手不敢動**:百岳資料海量,但「我適合哪一座」沒人告訴你 — 怕高估體能受傷,怕低估又浪費假期
- **健行筆記有資料但無 personalization**:它是百岳百科,不是個人化顧問
- **入山申請複雜**:玉山管處 / 雪管處 / 太管處 / 林務局 各自系統,搞不清楚要去哪個
- **季節 / 天氣 / 雪季限制**:同一座山春夏可走、冬季要技術裝備,新手不知道
- **海外工具(AllTrails / Komoot)無台灣百岳深度資料**

## 為什麼現在沒有對的工具

| 既有工具 | 為什麼不行 |
|----------|-----------|
| **健行筆記** | 最大台灣登山平台,**有資料但完全沒 AI personalization** |
| **百岳.net / Wikiloc** | 靜態山岳清單 |
| **AllTrails / Komoot** | 全球工具,**無台灣百岳深度內容** |
| **PTT / Dcard 詢問** | 看心情回覆,每次都重新問 |
| **ChatGPT 直接問** | 不知道你體能 + 不知道入山申請複雜度 + 不更新季節 |

**Gap 結構性**:健行筆記有完整資料但商業模式是廣告 + 社群,不會自己加 AI personalization 攻擊自家流量;海外工具不會本地化 266 座百岳。

## trailmatch 在做什麼

```
使用者 profile:
  經驗等級 / 體能 / 可用天數 / 出發城市 / 季節 /
  避免地形 / 高山症史 / 雪攀裝備 / 已有裝備 / 目標
        │
        ▼
  ┌──────────────────────────────────┐
  │ trailmatch.py 純函式硬條件過濾      │
  │  ✗ 難度 > 經驗上限                  │
  │  ✗ 天數 < 最少需要                  │
  │  ✗ 爬升 > 體能上限                  │
  │  ✗ 季節不在建議範圍                 │
  │  ✗ 高山症 + 高風險山                 │
  │  ✗ 含使用者避免地形                  │
  │  ✗ 冬季 3000m+ 無雪攀裝備            │
  │  ⇒ 通常 26 座剩 5-15 座進 LLM        │
  └────────────────┬─────────────────┘
                   │
                   ▼
  ┌──────────────────────────────────┐
  │ Claude 從通過清單挑 Top 3           │
  │  - why_match: 引用具體 profile      │
  │  - gear_tips: 季節 + 體能個人化     │
  │  - safety_warnings: 具體不空泛       │
  │  - party_advice: 結伴建議            │
  │  + overall_advice: 整合建議          │
  └────────────────┬─────────────────┘
                   │
                   ▼
        山岳推薦報告 markdown
        (登山者 / 教學中心參考)
```

3 個關鍵架構決策:

1. **`mountains_db.py` 結構化 corpus**(25 座):每座完整 metadata(高度 / 難度 1-5 / 天數 / 爬升 / 入山申請 / 季節 / 高山症風險 / 地形 / 出發城市 / 是否百岳 / 特殊注意)— 海外工具不可能複製這份在地化資料庫。
2. **純函式硬條件過濾不交給 LLM**:LLM 永不繞過硬條件(經驗不夠不能推進階百岳,即使 AI 覺得「應該可以」)。避免幻覺推薦危險路線。
3. **LLM 引用具體 profile 不空泛**:`why_match` 必須 cite 具體 profile 欄位(season / fitness / goals);`safety_warnings` 必須具體不寫「注意安全」這種廢話。

## 動作

### 純函式硬條件 baseline(免 API key)

```bash
python3 trailmatch.py samples/sample_input.json --today 2026-05-10 --no-ai
```

`samples/sample_input.json` 是初登者 / 中等體能 / 台北 / 春天 / 3 天 case。No-AI 模式只做硬條件 + 按 difficulty 降序排,**展示 baseline 但不夠智慧**(把「中級山非百岳」放 Top1,沒看到 user 想要「第一座百岳」)。

### 完整 AI 模式(需 API key)

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python3 trailmatch.py samples/sample_input.json --today 2026-05-10 --out report.md
```

AI 模式正確抓到使用者「第一座百岳」目標,Top 3 全部選**入門百岳**(合歡山主峰 / 石門山 / 合歡山東峰),並建議「合歡山三座連走」3 天 2 夜經典路線。

### 預先產出的 demo

`examples/sample_output.md` 是 AI 模式預期結果:
- **Top 1 = 合歡山主峰**(完美匹配:第一座百岳 + 拍美照 + 不太冷 三目標)
- **Top 2 = 石門山**(最簡單百岳,跟主峰連走 + 高山杜鵑拍照)
- **Top 3 = 合歡山東峰**(三座連走 + 稜線體驗)
- **綜合建議**:3 天 2 夜「合歡山三座連走」經典路線

`examples/sample_output_no_ai.md` 展示純函式 baseline,對比顯示 AI personalization 的價值。

## 收錄山岳(25 座 corpus,涵蓋台灣登山 80%+ 需求)

| 等級 | 代表山岳 |
|------|---------|
| **L1 郊山** | 七星山、大屯山 |
| **L2 入門百岳** | 石門山、合歡山主峰、合歡山東峰 + 中級山(石碇皇帝殿、鳶嘴山、阿里山特富野) |
| **L3 中級百岳** | 玉山前峰、雪山東峰、奇萊南華、合歡山北峰 |
| **L4 進階百岳** | 玉山主峰、雪山主峰、嘉明湖、北大武山、大霸尖山 |
| **L5 技術型** | 南湖大山、中央尖山、奇萊主北 |

(實際產品需擴充至 150+,含全 266 座百岳 + 各地中級山 + 接林務局 / 玉管處 / 太管處 入山申請即時 API。)

## 已驗證 smoke test

- ✅ Level 限制:beginner 不能通過 level 4 玉山主峰
- ✅ Expert + elite 體能 + 雪攀裝備:19/26 座通過(只有 L5 技術型 + 季節衝突剩餘)
- ✅ 冬季無雪攀裝備:合歡山主峰被拒(3000m+ 雪季條件)
- ✅ 高山症 + high risk 山:玉山主峰被拒
- ✅ Sample case:beginner / medium 體能 / 春 / 避免 vertical → 通過 7 座**正確包含 3 座入門百岳**(石門山 / 合歡主峰 / 合歡東峰),正確排除 玉山主峰(難度) + 鳶嘴山(vertical)

## 真正產品要有但 prototype 沒做

- **接入山申請 API**:玉管處 / 雪管處 / 太管處 / 林務局 即時抽籤結果 + 候補
- **天氣即時整合**:中央氣象局山區天氣 API,出發前 7 天天氣推送
- **動態山屋床位查詢**:三六九 / 排雲 / 嘉明湖 山屋床位
- **完整 150+ 山岳 corpus**:從 25 擴充到全台百岳 + 熱門中級山 + 私房路線
- **登山日誌 / GPX 紀錄**:user 上傳走過的山自動進化 profile(從 beginner → intermediate)
- **同伴媒合**:相同 profile + 相同山的人配對(雙邊 marketplace)
- **裝備租借 partner**:跟登山補給站 / Go Pro 合作租借
- **山屋訂房代辦**:複雜申請流程一鍵代申請(成功率 trick + 候補通知)
- **緊急定位 / SOS**:配合 Garmin inReach / 衛星電話租借
- **PDF 行程表 + GPX 下載**

## 商業模式

| 方案 | 月費 | 對象 |
|------|------|------|
| **Free** | 月 3 次推薦 | 試用 / 新手 |
| **登山者 Pro** | **NT$149 / 月** | 登山愛好者 |
| **Plus 路況訂閱** | NT$299 / 月 | + 即時天氣推送 + 山屋床位通知 |
| **登山旅行社 B2B** | NT$2,000 / 月 | 客製團體行程規劃 |
| **戶外用品聯盟** | 5-10% 佣金 | Go Pro 登山補給站 / 大自然 / 山林好朋友 |

### WTP 計算

- 登山客一趟成本 NT$5K-30K(交通 / 山屋 / 裝備折舊 / 嚮導 / 食物)
- 一次選錯路線(超出體能 / 季節錯) = 提早撤退 = 損失整趟成本
- vs Pro NT$149/月 = **單次避免錯誤即年費回本**
- Garmin inReach 等 SOS 裝置月費 NT$300+,登山客已習慣付月費

### TAM

- 台灣登山人口估 200 萬 + 百岳活躍登山客 30-50 萬
- 取 5% 滲透 = 1.5-2.5 萬人 × NT$149 = **月 NT$220-370 萬 MRR / 年 NT$2,640-4,440 萬 ARR**
- 加 B2B 旅行社 + 戶外用品聯盟 → 翻倍

## 風險評估

| 風險 | 評估 | 緩解 |
|------|------|------|
| **健行筆記自己加 AI** | 中~高 | 他們社群 + 廣告商模有結構性衝突;搶心占率 |
| **AI 誤判導致登山事故** | 高 | 純函式硬條件不繞過;報告寫明免責;鼓勵結伴 |
| **海外 AllTrails 進台灣** | 低 | 在地化護城河深(266 百岳 + 入山申請 + 山屋床位) |
| **入山申請成功率不穩** | 中 | 提供候補通知 + 替代路線推薦 |
| **季節變動超出 corpus 預設** | 中 | corpus 季節 metadata 季度 update + 接氣象局 API |
| **登山者覺得「我自己會選」** | 中 | 免費 3 次/月 hook + 初登者 onboarding 流程 |
