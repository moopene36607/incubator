# vetnote

**台灣獸醫診所中文 SOAP 病歷草稿 AI 助手。**

獸醫師講一段話或填一個 form,30 秒內變成完整的 4 段 SOAP 病歷草稿(主訴 / 客觀 / 評估 / 計畫)。把每天 30-50 份病歷的 2-3 小時文書時間壓到 30 分鐘的「審閱簽核」。

---

## 痛點

台灣 **2,600 家動物醫院 / 診所** 的獸醫師每天最大時間漏斗不是診療,是病歷。

> **「每天打 SOAP 打到手軟,同樣的主訴打幾十遍。」** — PTT VetMed 板常見討論

> **「實習最痛苦的是病歷,要寫到 10 點才能走。」** — Dcard 獸醫系版

獸醫師每天 20-50 個 case,每個 case 需要寫:
- **S** (Subjective):主訴、病史、OPQRST、飼主敘述
- **O** (Objective):TPR、體重、PE、實驗室、影像
- **A** (Assessment):鑑別診斷、工作診斷
- **P** (Plan):進一步檢查、治療、飼主衛教、回診

每份病歷 5-15 分鐘 × 30 份/天 = **每天 2.5-7.5 小時純文書**。
小型動物醫院 (1-3 人) 沒有專屬書記人員,獸醫師白天看診、晚上補病歷,實習生更慘。

## vetnote 在做什麼

```
獸醫師輸入(語音 or 結構化 form)
  - 病患資料 (品種 / 年齡 / 體重 / 晶片)
  - 主訴大類 (10 類預設)
  - S 段資訊 (主訴 / 病史)
  - O 段資料 (TPR / PE / 實驗室 / 影像)
        │
        ▼
  ┌──────────────────────────────────┐
  │ 純函式組裝病歷標頭 + 簽核欄        │
  │  - 病患基本資料表                  │
  │  - 就診資訊 (日期 / 類型 / 主訴大類) │
  │  - 簽核欄 (獸醫師審閱必填)          │
  │  - AI 草稿免責聲明                 │
  └────────────────┬─────────────────┘
                   │
                   ▼
  ┌──────────────────────────────────┐
  │ Claude 撰寫 SOAP 4 段             │
  │  - S/O/A/P 結構化輸出             │
  │  - 主訴大類 grounding (10 類預設)  │
  │  - 鑑別診斷由可能性高到低         │
  │  - **絕不開具體劑量** (獸醫師職責) │
  │  - **絕不下確定診斷** (只能 dx)   │
  └────────────────┬─────────────────┘
                   │
                   ▼
        完整 markdown SOAP 病歷草稿
        (獸醫師審閱 → 簽核 → PIMS)
```

## 三大架構決策

1. **`soap_categories.py` 純資料 grounding**:10 種常見主訴大類,每類附 「O 段必記欄位」、「typical_workup」、「treatment 類別」。Prompt 帶這些 ground truth,AI 不會漏寫關鍵欄位。
2. **AI 不開劑量、不下定論**:System prompt 嚴格規定「治療只寫類別 (例:止吐劑、靜脈輸液),不寫具體藥名與 mg/kg」、「A 段必須使用『考慮』『待排除』『初步診斷』」。獸醫法規定診斷與處方是獸醫師專屬,我們只做草稿。
3. **三道關 = 法律免責**:獸醫師審閱 → 簽核 → 輸入 PIMS,病歷尾段強制顯示免責聲明。AI 草稿 ≠ 病歷正本。

## 動作

### 不開 AI(骨架版,免 API key)

```bash
python3 vetnote.py samples/sample_input.json --no-ai
```

骨架版會渲染病患標頭 + 就診資訊 + 簽核欄,但 SOAP 4 段是 placeholder,留給獸醫師手填。已驗證 — 立即可跑。

### AI 模式(需 API key)

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python3 vetnote.py samples/sample_input.json --out record.md
```

AI 根據 sample 案 (4 歲米克斯急性嘔吐 + 中度脫水) 產出完整 SOAP 4 段,包含 5 項鑑別診斷、工作診斷、4 項進一步檢查、6 類治療、4 條飼主衛教、回診計畫。

### 預先產出的 demo

`examples/sample_output.md` 是完整 SOAP 病歷草稿,展示典型急診 case (急性嘔吐 + 中度脫水) 的全貌:
- S 段:24h 嘔吐演變 + 2 週前誤食史 + 疫苗預防醫療紀錄
- O 段:TPR 39.2/108/28 + 體重下降 0.5kg + PE 6 系統 + AXR + 完整 lab(CBC/biochem/electrolytes/SNAP)
- A 段:5 項鑑別診斷 + 工作診斷論述
- P 段:4 項 workup + 6 類治療(只寫類別)+ 4 條飼主衛教 + 回診計畫

## 真正產品要有但 prototype 沒做

- **語音輸入整合**:獸醫看診邊講邊產出 (Whisper 繁中已商用)
- **PIMS 整合 (毛孩管家 / 獸易通 / 寶力)**:草稿直接寫入 PIMS,不要複製貼上
- **DICOM 影像 vision 分析**:X 光直接送入 vision API 讀取,不需獸醫師打字描述
- **完整主訴大類擴充**:30+ 類涵蓋眼科 / 神經 / 心臟 / 內分泌 / 腫瘤 / 行為
- **歷次病歷 RAG**:同一病患歷次 SOAP 自動帶入 context,延續性追蹤
- **趨勢圖表**:體重 / 血糖 / 腎指數歷次變化視覺化
- **多獸醫共享病歷模板**:單位內統一格式
- **疫苗 / 蝨蚤 / 心絲蟲 提醒系統**:從病歷自動推斷下次到期日 → LINE 推飼主
- **馬科 / 牛科 / 禽科**:大動物獸醫市場(目前犬貓限定)

### WTP 計算

- 獸醫師寫 1 份 SOAP 5-15 分鐘 = 平均 10 分鐘
- 每天 30 份 × 10 分鐘 = **5 小時/天文書時間**
- 獸醫師時薪 NT$800-1,500
- vetnote 把它壓到 1.5 小時(審閱)= **省 3.5 小時/天 × NT$1,000 = NT$3,500/天**
- 月省 NT$77,000 vs 月費 NT$1,800 = **42 倍 ROI**(實際遠超,因為這時間可換成多看 case 直接增加營收)

對比 VetSnap USD $99 = NT$3,200,**vetnote NT$1,800 有 44% 價格優勢**。

### TAM

- 2,600 家動物醫院 × NT$1,800 = 月 NT$468 萬全市場
- 5% 滲透 = 130 家 × NT$1,800 = **月 NT$23.4 萬 MRR / 年 NT$280 萬 ARR**
- 加上獸醫個人版 (5,500+ 註冊獸醫師) × NT$1,200 個人版 → TAM 翻倍
- 海外橫移日本 / 韓國 / 香港繁中市場 → TAM x4

## 風險評估

| 風險 | 評估 | 緩解 |
|------|------|------|
| **VetSnap / Scribenote 推繁中版** | 中 | 英文市場仍未飽和,繁中 ROI 對他們不夠;先佔在地市場 |
| **獸易通 / 毛孩管家 PIMS 自己出 AI** | 中~高 | PIMS 廠商開發慢、且大家都知道做 AI 不容易;搶占心占率 |
| **AI 幻覺出錯藥物 / 劑量** | 高 | System prompt 嚴禁開具體藥名劑量 + 獸醫師審閱 final gate + 草稿明確標注「需簽署確認」 |
| **獸醫師法 / 動保法 規範 AI 病歷** | 中 | 主動定位「協助起草」非「代替診斷」;書面審閱簽署是法律 final gate |
| **小診所對 AI 抗拒** | 中 | NT$1,200 個人版,獸醫師個人決策不需診所採購 |
| **資料隱私 / HIPAA-equivalent** | 中 | 病歷不存伺服器(stateless API call)+ 飼主姓名以代稱;企業版可走 self-host |
