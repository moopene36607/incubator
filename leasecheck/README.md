# leasecheck(租約掃描)

**台灣住宅租賃合約 AI 審約助手 — 一份合約 30 秒抓出霸王條款 + 給出談判話術。**

LLM 從合約原文精確抽取每條條款 → 純函式比對 corpus + 純函式算風險分數 → AI 寫房客易讀的解釋與談判話術。

把「800 字密密麻麻的合約看完不知道哪邊有問題」變成「重點 3 條紅色條款先談」。

---

## 痛點

全台租屋族 **200+ 萬人**,每年新合約 **50-80 萬份**,但 99% 簽約者**不會看條款**。

> **「房東給我的合約寫押金 3 個月、違約金 2 個月、不能報稅 — 我覺得怪怪的但又不敢跟房東吵。」** — Dcard 租屋板

> **「住了半年發現修繕全自己付,馬桶壞掉房東叫我自己處理,問律師才知道合約有問題,但已經來不及了。」** — PTT home-sale

> **「想諮詢律師審約 NT$3,000-5,000,小資族真的負擔不起。」** — FB 租屋自救會

具體痛點:

- 房東提供的「自製合約」常**抄到 5 年前的版本**,根本不知道內政部 2017 應記載 / 不得記載事項
- 常見坑:押金 3-6 個月(違法)、違約金 2 個月(違法)、修繕全推房客(違法)、強制續約(違法)、不得申報所得稅(違法)、房東隨時進入(違法)
- 律師審約 NT$3-5K,**對月薪 28-35K 的租屋族太貴**
- 房客 **怕跟房東關係搞僵**,即使覺得有問題也不敢提
- 簽下去就是 1 年,**有問題只能吃虧**(押金被扣 / 修繕自付 / 漲房租被迫接受)
- 內政部「住宅租賃定型化契約應記載及不得記載事項」公開可下載,**但沒人把它做成 AI 審約 SaaS**

## 為什麼現在沒有對的工具

| 既有工具 | 為什麼不行 |
|----------|-----------|
| **崔媽媽 / 法扶基金會 諮詢專線** | 免費但要排隊;只能口頭諮詢無書面審約;無法直接拿合約上去掃 |
| **律師審約 (NT$3-5K)** | 太貴;1 份合約值不值得花 5K 房客很猶豫 |
| **DoNotPay / Robinhood (海外)** | 完全不認識台灣租賃條例 + 民法 + 應記載事項;條款根本不對 |
| **LegalBank / 法源 / 法源編譯** | 只是法條搜尋,無法審契約 |
| **ChatGPT / Claude 直接問** | 容易幻覺(編造法律條文);不知道台灣具體條例;不能引用具體法源 |
| **591 / 樂屋網 自製範本** | 是空白範本,不能審你已經拿到的合約 |
| **房客自己 Google** | 30 條條款 × 每條花 10 分鐘查 = 5 小時,且很容易看漏 |

**Gap 結構性**:中文租賃法規開放、租屋族痛點明確、律師審約太貴、海外工具不在地化 — **典型 niche-SaaS 機會**。Google 搜尋「台灣 租屋合約 AI 審約」零 SaaS 結果。

## leasecheck 在做什麼

```
用戶上傳租賃合約 (.txt / OCR PDF)
        │
        ▼
  ┌──────────────────────────────────────────────────────────┐
  │ Claude 結構化抽取(每條條款 → 對應 corpus code):           │
  │   - 押金條款:「押金為新台幣肆萬伍仟元(三個月租金)」          │
  │     → DEPOSIT_OVER_2MTH (red)                              │
  │   - 修繕條款:「修繕費用,概由租客自行負擔」                   │
  │     → REPAIR_ALL_ON_TENANT (red)                           │
  │   ... 共比對 corpus 20+ 條典型條款                          │
  │ **LLM 嚴禁編造合約沒寫的條款**                              │
  └────────────────┬─────────────────────────────────────────┘
                   │
                   ▼
  ┌──────────────────────────────────────────────────────────┐
  │ 純函式 assess_risk:                                       │
  │   - 加總每條 hit 的 severity_score → risk_score            │
  │   - 切檔 LOW / MEDIUM / HIGH / **CRITICAL**                │
  │ **LLM 永不算分**                                            │
  └────────────────┬─────────────────────────────────────────┘
                   │
                   ▼
  ┌──────────────────────────────────────────────────────────┐
  │ Claude 為每條 red / yellow 條款寫:                         │
  │   - explanation_for_renter (50-100 字房客易讀)             │
  │   - negotiation_script (1-2 句房客可原話跟房東講的台詞)     │
  │ + 整體建議(本份合約該優先談 3 件事)                        │
  └────────────────┬─────────────────────────────────────────┘
                   │
                   ▼
        markdown 報告(風險摘要 + 條款逐條解析 + 談判話術)
```

3 個關鍵架構決策:

1. **LLM 抽取 + 純函式評分**:合約原文是非結構化的,LLM 善於「精確找出對應條款的句子」;但風險分數計算 100% 純函式(`analyzer.py:assess_risk`),**LLM 永不算分**。
2. **corpus 預設 20+ 條 + 法源**:每條 clause 都附 `legal_basis`(內政部不得記載事項第 N 點 / 民法第 X 條 / 租賃住宅市場發展條例第 Y 條)— 房客拿著這份報告跟房東談時可以**有所本**。
3. **不勸放棄 / 不勸提告**:negotiation_script 永遠以「修改條款」為主,**避免 escalate 衝突**。重大爭議引導到崔媽媽 / 法扶基金會,而非 leasecheck 自己處理。

## 動作

### 純函式 keyword 比對(免 API key)

```bash
python3 leasecheck.py samples/lease.txt --no-ai --out output.md
```

純函式 fallback 模式:用 keyword 比對 corpus 找出可能命中的條款。 covers ~70-80% 的常見條款,但精確抽句、談判話術需開 AI 模式。

### 完整 AI 模式(需 API key)

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python3 leasecheck.py samples/lease.txt --out output.md
```

AI 模式:Claude 從合約抽出**精確的條款原文**(而非 keyword 周圍的 snippet)+ 寫每條條款的房客易讀解釋 + 談判話術。

### 預先產出的 demo

`examples/sample_output.md` 是完整 AI 模式報告,展示 10 紅 + 5 黃條款 + 整體建議 + 每條談判話術。

`examples/sample_output_no_ai.md` 是純函式模式輸出,證明免 API key 也能用。

## 已驗證 smoke test

- ✅ 預設 sample(刻意設計 14 條霸王條款的合約) → CRITICAL(score 100, 10 red + 3 yellow + 1 green)
- ✅ 乾淨合約(只含安全條款) → LOW(score 0, 0 red)
- ✅ 1 紅 + 1 黃 → MEDIUM(score 33)
- ✅ 空白合約 → LOW(0 hits)
- ✅ assess_risk 不重複加分(同 code 多次出現算一次)
- ✅ merge_hits 優先 LLM 結果,keyword 補上 LLM 漏掉的
- ✅ 中文空白容差(「7 日內遷離」 / 「7日內遷離」都能匹配)
- ✅ keyword 不會 false-positive 到綠色 / 安全條款(「房客負擔過失損害」 不會誤觸 REPAIR_ALL_ON_TENANT)

## 專案結構

```
leasecheck/
├── README.md
├── leasecheck.py         # CLI(讀檔 + 純函式 keyword + LLM 抽取 + 渲染報告)
├── analyzer.py           # 100% 純函式:keyword extract + 風險評分
├── clauses_db.py         # 20+ 條台灣租賃常見條款 corpus + 法源 + 嚴重度權重
├── samples/
│   └── lease.txt         # 刻意設計含 14 條霸王條款的台灣典型套房租約
├── examples/
│   ├── sample_output.md          # AI 模式完整報告(10 red + 5 yellow)

```

純函式部分零外部依賴(stdlib only)。依賴僅 `anthropic`(AI 模式才需要)。

## 真正產品要有但 prototype 沒做

- **PDF / 圖片 OCR**:用戶手機拍房東給的合約 PDF,Vision API 讀文字 + leasecheck 審約
- **手機端 LINE Bot**:LINE 上傳合約 → 30 秒回 risk 摘要 + 紅色條款警示
- **修約建議直接生成 PDF**:房客 fork 合約 + 標出建議修改處 → 列印給房東看
- **連結房東「應記載 / 不得記載事項」原文 PDF**:每條 hit 點進去看內政部官方公告
- **崔媽媽基金會 / 法扶整合**:risk 太高直接幫用戶預約諮詢時段
- **歷史合約資料庫**:用戶歷次審過的合約資料(房東 X / 物件 Y)幫下次決策
- **房東黑名單眾包**:重複出現霸王條款的房東 hash 化匯總(隱私 / 法律問題需審慎)
- **多模態 vision**:手寫 / 手畫圖 標註的合約變更也能讀

## 商業模式

| 方案 | 月費 / 單次費 | 對象 |
|------|--------------|------|
| **Free** | 月 1 份審約 | 試用 |
| **Solo** | **NT$99 / 月** | 單份合約簽前審閱(學生 / 租屋族) |
| **Pay-per-use** | NT$199 / 份 | 不訂閱,簽前單次審 |
| **Pro** | NT$299 / 月 | 包修約建議 + LINE Bot + 法扶預約 |
| **B2B 房仲** | NT$1,999 / 月 | 永慶 / 信義 / 中信 房仲業務員幫客戶審 |
| **B2B 公益** | 免費(由政府或基金會贊助) | 崔媽媽 / 法扶基金會合作 |

### WTP 計算

- 律師審約 NT$3,000-5,000 vs leasecheck NT$199 / 份 = **15-25x 便宜**
- 一條霸王條款被擋下(押金多收 1 個月 NT$15K + 違約金多 1 個月 NT$15K) = **直接省 NT$30K**
- B2B 房仲:每個業務員月成交 1-2 件,每筆服務費 NT$1.5-3 萬,leasecheck 月費 NT$1,999 = 0.5-1% 成本 → 高 ROI

### TAM

- 全台租屋族 200 萬人,每年新合約 50-80 萬份
- 取 5% 滲透率(c2c) = 25,000-40,000 人 × 平均 NT$199/年(混 Free + Solo + Pay-per-use)= **年 NT$500-800 萬 ARR**
- 加 Pro 訂閱 + B2B 房仲 + 政府合作 → 年 ARR **NT$2,000-4,000 萬**
- 横移到日本 / 韓國 / 新加坡 (中港的住宅租賃條款結構雖然不同,但商業合約有類似痛點) → 翻倍

## 風險評估

| 風險 | 評估 | 緩解 |
|------|------|------|
| **AI 抽錯 / 漏抽條款** | 中-高 | 純函式 keyword fallback + LLM 雙保險;隨時可開源 corpus 接受社群修正 |
| **被定位成「教用戶跟房東吵架」反感** | 中 | UI 強調「協助溝通」非「對抗」;negotiation_script 都是建設性語氣;明確標出「合理 / 不合理」分檔 |
| **對房東 / 房仲不友善** | 中 | 明確說「不勸提告 / 不勸放棄」;對 B2B 房仲推「快速審閱」價值,讓房仲變幫手 |
| **法律意見責任** | 高 | UI / report 多處明確 disclaimer「本工具不是法律意見」+ 重大爭議引導到法扶 / 律師 |
| **房東提供的合約是 PDF 圖片** | 中 | Pro 版加 OCR(Vision API);Free 版要求純文字輸入 |
| **崔媽媽 / 法扶 自己做** | 低-中 | 公益機構通常人手不足且開發慢;leasecheck 早期可主動合作而非競爭 |
| **個資 / 合約隱私** | 高 | stateless API call;合約不上雲;企業版可走 self-host LLM |
| **房屋 / 商辦 / 倉庫等租約結構不同** | 中 | 第 1 版只做住宅;商辦版另開 corpus(內政部商業租賃定型化契約) |
